# R13: 리포트 품질 개선 (코칭 톤 + 씬 중복 + UI)

## 문제 분석

### 1. 코칭 톤 (P13)
- "사회적 증거", "pain_point", "긴급성 요소" → 마케팅 전문 용어 직접 노출
- 사용자(마케터/크리에이터)에게도 불친절
- **해결**: P13 프롬프트에 톤 가이드 추가
  - 전문 용어 → 쉬운 말로 풀어쓰기
  - 코칭 톤 유지 (존댓말, 구체적, 실행 가능)
  - 예: "사회적 증거 추가" → "실제 후기나 반응 장면을 넣으면 신뢰도가 올라갑니다"

### 2. 씬 중복 (대본↔씬 매핑)
- 씬이 1~2초 단위로 너무 잘게 쪼개짐 (P3 scene detection)
- 대본 블록 1개에 씬 3~7개 매핑 → 같은 대사가 여러 씬 카드에 반복
- **해결 방향 A**: 프론트에서 같은 블록에 속한 씬들을 그룹핑 (UI 레벨)
- **해결 방향 B**: P3 씬 병합 임계값 조정 (백엔드 레벨)
- **채택: A** — 백엔드 변경 없이 프론트에서 해결. 씬 카드를 블록 기준으로 그룹핑하고, 그룹 내 씬은 썸네일 스트립으로 표시

### 3. UI 개선
- 씬 카드 과다 → 기본 접힘 + "더보기"
- 모바일 텍스트 가독성 (줄간격, 폰트)
- Hook/Body/CTA 코칭 카드 아코디언
