# WakaLab — Product Strategy & PRD

> 마케터의 마이크로 AX 연구소
> 최초 작성: 2026-02-21

---

## 1. 비전

### 한 줄 정의
**WakaLab = 마케터의 업무를 하나씩 AI로 전환해주는 연구소**

### 왜 "Lab"인가
- 연구하고, 실험하고, 검증된 것만 출시한다
- 마케터가 함께 만들어가는 플랫폼 (수요 기반 개발)
- 도구를 파는 게 아니라, **마케터의 AX 전환 자체를 돕는 것**

### 포지셔닝
```
기존 SaaS: "우리 도구 쓰세요" (공급자 중심)
WakaLab:   "뭐가 필요하세요? 만들어드립니다" (수요자 중심)
```

### 핵심 철학
- **마케터가 원하는 것만 만든다** — 수요 없으면 개발 없음
- **성과를 올려주는 것만 만든다** — CTR, ROAS, 제작 효율
- **하나의 플랫폼, 모듈로 확장** — 운영 리소스 최소화

## 2. 수요 기반 개발 시스템 (Lab Request)

### 개요
마케터가 "이런 AI 도구 있었으면" → 요청 점수가 쌓임 → 100점 달성 시 개발 시작

### 점수 체계

| 행동 | 점수 | 의미 |
|------|------|------|
| 무료 투표 (관심 표현) | 1점 | "나도 필요해" |
| 쿠폰 구매 (선결제) | 100점 | "돈 내고 쓸게, 빨리 만들어줘" |
| 독점 사용권 구매 | 1,000점 | "우리만 쓸 수 있게 만들어줘" |

### 규칙
- **100점 이상** → 개발 대기열 진입
- **동시 개발 도구: 최대 1~2개** (팀 규모 고려)
- 쿠폰 구매자 → 출시 후 무료 이용 or 할인 (얼리버드 혜택)
- 독점 구매자 → 일정 기간 독점 후 일반 공개 (or 영구 독점은 별도 협의)

### 단, 이 시스템은 2단계에서 도입
1단계에서는 첫 도구로 마케터를 모으는 게 먼저.

## 3. 성장 단계

### Stage 1 — 첫 도구로 유입 (현재 ~ 3개월)

**목표:** 영상 분석/진단 도구 완성 → 마케터 유입 → 니즈 수집

| 항목 | 내용 |
|------|------|
| 핵심 도구 | 숏폼 마케팅 영상 분석·진단 |
| 타겟 | 퍼포먼스 마케터, 이커머스 마케터, 광고 대행사 |
| 수익 | 프리미엄 (기본 무료 + 고급 기능 유료) |
| KPI | 가입자 수, 분석 건수, 유료 전환율 |
| 마케팅 | 분석 결과 콘텐츠화 (LinkedIn, X, 블로그) |

**이 단계에서 검증할 것:**
- 마케터가 실제로 돈을 내고 쓰는가?
- 어떤 기능에 가장 반응하는가?
- "이런 것도 되나요?" 질문 수집 → 다음 도구 후보

### Stage 2 — 커뮤니티 + 투표 시스템 (3~6개월)

**목표:** Lab Request 시스템 오픈 → 다음 도구 수요 검증

| 항목 | 내용 |
|------|------|
| 신규 기능 | Lab Request (투표/쿠폰/독점) |
| 커뮤니티 | 마케터 피드백 채널 (디스코드 or 슬랙 or 자체) |
| 수익 | 구독 + 쿠폰 선결제 |
| KPI | 투표 참여율, 쿠폰 판매 수, 커뮤니티 활성도 |

### Stage 3 — 멀티 도구 플랫폼 (6개월~)

**목표:** 검증된 도구 추가 → "마케터의 AI 도구 모음" 포지셔닝

| 항목 | 내용 |
|------|------|
| 도구 | 2~3개 (수요 검증된 것만) |
| 수익 | 통합 구독 + 맞춤 솔루션 |
| KPI | 도구별 MAU, 통합 구독 전환율, 리텐션 |

## 4. Stage 1 PRD — 숏폼 영상 분석·진단

### 4.1 제품 개요

| 항목 | 내용 |
|------|------|
| 기능명 | 영상 분석·진단 (Video Diagnosis) |
| 한 줄 | "숏폼 영상 넣으면 → 왜 잘 됐는지 + 어떻게 고칠지" |
| 타겟 유저 | 숏폼 광고 운영하는 퍼포먼스 마케터 |
| 핵심 가치 | 감이 아닌 데이터로 영상 피드백 → 제작비 절감 + 성과 향상 |

### 4.2 사용자 흐름

```
① 회원가입/로그인 (Google OAuth)
     ↓
② 영상 업로드
   - 형식 선택: [Auto] / 진행자형 / 캡션형 / 데모형 / ...
   - 의도 선택: [Auto] / 커머스 / 정보전달 / 브랜딩 / ...
     ↓
③ 분석 중 (1~3분)
   - 진행 상태 표시
     ↓
④ 리포트 확인
   - 영상 플레이어 + 4탭 분석 리포트
     ↓
⑤ (유료) 비교 분석 / PDF 다운로드 / 기획안 생성
```

### 4.3 리포트 구성

#### 공통: 영상 플레이어
- 좌측 고정, 씬/소구 클릭 시 해당 시점 자동 이동
- 5차원 점수 차트 (플레이어 하단)

#### 탭 1: 소구 흐름도 (Appeal Flow) — "발견"
> 영상의 설득 구조를 한 눈에

- 씬 역할별 색상 타임라인 (hook=빨강, demo=파랑, proof=초록, cta=주황)
- 소구 포인트 마커 (🎬 비주얼 / 📝 캡션 구분)
- 소구 공백 구간 하이라이트 (⚠️ 이탈 위험)
- 시각 자극도 커브를 배경 레이어로 (Stage 1 후반)

#### 탭 2: 씬 카드 (Scene Cards) — "근거"
> 왜 이 장면이 효과적인가

- 씬별 카드: 역할 뱃지, 시간, 집중도 바
- 소구 포인트 상세 (타입, 주장, 비주얼 기법)
- 카드 클릭 → 영상 해당 시점 이동

#### 탭 3: 진단서 (Diagnosis) — "처방"
> 강점/약점 + 구체적 수정 가이드

- ✅ 강점 리스트
- ⚠️ 약점 리스트
- 💊 처방 (우선순위, severity 색상)
- 🩺 구간별 상세 진단 (시간 클릭 → 영상 이동)

#### 탭 4: 아트 디렉션 — "스타일 가이드"
> 제작팀 전달용 비주얼 레퍼런스

- 톤 & 매너
- 타이포그래피 (헤딩/본문 폰트)
- 컬러 팔레트 (칩 시각화)
- 구도/레이아웃 규칙

### 4.4 과금 모델

| 티어 | 가격 | 내용 |
|------|------|------|
| Free | ₩0 | 월 3건 분석, 기본 리포트 (소구 흐름도 + 진단서) |
| Pro | ₩49,000/월 | 무제한 분석, 전체 리포트, PDF 다운로드 |
| Team | ₩149,000/월 | Pro + 비교 분석, 팀 공유, 기획안 생성 |

*가격은 검증 후 조정. 초기에는 Pro 무료 체험으로 유입 유도.*

### 4.5 기술 스택

| 항목 | 기술 | 비고 |
|------|------|------|
| 프론트엔드 | React + Vite + Tailwind | Cloudflare Pages |
| 백엔드 API | FastAPI | Cloud Run (asia-northeast3) |
| 분석 엔진 | video-analyzer (자체) | Python, 7-Phase 파이프라인 |
| AI | Gemini 2.5 Flash | Tier 3, 분석당 ~₩150 |
| STT | Soniox (배포) / Whisper (로컬) | Phase 0 |
| DB | Supabase | Auth + PostgreSQL |
| 스토리지 | Cloudflare R2 | 영상 업로드 |
| 도메인 | 미정 | wakalab.io 후보 |

### 4.6 분석 파이프라인

```
Phase 0   STT (Soniox) → voice/caption/silent 판별
Phase 0.1 Style Classification (Gemini Lite) → 형식 × 의도
Phase 0.5 SceneDetect → 컷 타임스탬프
Phase 1   OpenCV → 시각 정량 분석
Phase 2   Gemini Flash → OCR + 시각 정성 분석
Phase 2.5 시계열 분석 → 컷 리듬, 어텐션 커브
Phase 3   씬 통합 (로컬, ₩0)
Phase 4   Gemini Flash → 영상 의미 분석 (스타일별 프롬프트)
Phase 5   씬 머지 (로컬, ₩0)
Phase 6   레시피 빌드 (로컬, ₩0)
Phase 7   통합 진단 + 처방 (로컬, ₩0)
```

### 4.7 5차원 분석 체계 (내부)

| 차원 | 측정 내용 | 데이터 소스 |
|------|----------|-----------|
| 시각적 자극도 | 컷 밀도, 씬 다양성, 어텐션 피크 | OpenCV + SceneDetect |
| 설득 밀도 | 소구 유무/강도, 비주얼 vs 캡션 | Gemini Phase 4 |
| 정보 밀도 | 텍스트양, 내레이션 속도, 핵심 정보 비율 | STT + OCR |
| 편집 리듬 | 컷 간격 분포, 리듬 패턴 | SceneDetect |
| 청각 자극도 | BGM, 음성 유형, 효과음, 레이어 | Gemini Phase 4 |

### 4.8 스타일별 분석 분기

**형식 9종:** 진행자 / UGC / 캡션텍스트 / 데모 / ASMR / 비교 / 나열 / 스토리 / 엔터
**의도 4종:** 커머스 / 정보전달 / 브랜딩 / 엔터테인먼트

> 같은 현상도 스타일에 따라 처방이 반대 — WakaLab의 핵심 차별점
> 예: 컷 밀도 0.3/초 → 커머스면 ⚠️"느림" / UGC면 ✅"적절"

## 5. Stage 1 마일스톤

### M1: MVP 완성 (2주)
- [ ] 리포트 페이지 완성 (현재 데모 → 실데이터 연동)
- [ ] 업로드 → 분석 → 리포트 E2E 동작
- [ ] 형식/의도 Auto 분류 결과 리포트에 표시
- [ ] 랜딩 페이지 (서비스 소개 + CTA)

### M2: 베타 오픈 (4주)
- [ ] Google OAuth 로그인
- [ ] 분석 히스토리 대시보드
- [ ] 형식/의도 수동 선택 UI
- [ ] 비교 분석 (A vs B) 기본 버전
- [ ] 베타 테스터 10명 모집

### M3: 유료화 (8주)
- [ ] 과금 시스템 (포트원 or Stripe)
- [ ] Free / Pro 티어 분리
- [ ] PDF 리포트 다운로드
- [ ] 니즈 수집 폼 ("다음에 어떤 도구 원하세요?")

### M4: 커뮤니티 (12주)
- [ ] Lab Request 투표 시스템
- [ ] 쿠폰/독점 구매 기능
- [ ] 다음 도구 후보 공개 + 투표 시작

## 6. 팀 & 역할

| 역할 | 담당 | 주요 업무 |
|------|------|----------|
| 대표 / PO | 김태영 | 방향 결정, 마케팅, 고객 소통 |
| 기획 | (팀원 1) | UX 설계, 요구사항 정리, QA |
| 개발 | (팀원 2) | 프론트/백엔드 구현 |
| AI / 인프라 | 브릿지 + 에이전트팀 | 분석 엔진, 배포, 운영 자동화 |

### 운영 원칙
- 동시 개발 도구: **최대 1~2개**
- 새 도구 추가 = 기존 플랫폼에 모듈 추가 (별도 인프라 ❌)
- AI 에이전트로 운영 자동화 (모니터링, CS, 에러 대응)
