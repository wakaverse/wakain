## 0. 핵심 원칙

**이 랜딩의 목표는 하나: "URL을 넣게 만드는 것"**

- 기능을 나열하지 말고, 가치를 보여줘라
- 마케터가 쓰는 언어로 말해라 (Spike, Hack 같은 내부 용어 금지)
- 텍스트보다 실제 분석 결과 스크린샷이 10배 강하다
- 모든 섹션은 "URL을 넣어보고 싶다"는 충동을 만드는 데 기여해야 한다

**타겟:** 마케팅 에이전시 AE (Account Executive)
- 클라이언트한테 "다음 영상은 이렇게 만들겠습니다" 제안서를 써야 하는 사람
- "경쟁사 영상은 잘 되는데 우리 건 왜 안 되지?"가 매일의 고민

**톤:** 전문적이되 무겁지 않게. B2B SaaS의 신뢰감 + 스타트업의 날카로움.
**현재 톤(다크 히어로 + 클린 화이트 바디)은 유지해도 좋음.**

## 3. 섹션1: 히어로

### 레이아웃
좌측: 텍스트 + URL 입력창
우측: 와카랩 분석 결과 스크린샷 (통합 타임라인 캡처)

### 카피

**상단 뱃지:** `COMMERCE VIDEO INTELLIGENCE` (현재 유지)

**헤드라인:**
```
경쟁사 숏폼,
왜 잘 되는지 알려드립니다
```

**서브헤드:**
```
영상의 구조를 컷 단위로 해부하고,
내 영상과 비교해서, 개선 방향까지 코칭해드립니다
```

**URL 입력창 (히어로에 직접 배치):**
- 플레이스홀더: `분석할 영상 URL을 붙여넣어 보세요`
- 버튼: `무료 분석 시작 →`
- 하단 작은 텍스트: `틱톡, 인스타 릴스, 유튜브 숏츠 지원 · 회원가입 없이 체험`
- URL 넣으면 → 가입 페이지로 이동 (URL은 세션에 저장, 가입 후 바로 분석 시작)

**우측 이미지:**
- 와카랩 통합 타임라인 스크린샷 (딸기 푸딩 또는 황금고구마 분석 결과)
- 브라우저 프레임(mockup)으로 감싸서 "실제 서비스 화면"임을 보여줌
- 살짝 기울어진 각도(perspective tilt)로 입체감 추가
- 선택: 스크린샷 위에 핵심 수치 몇 개를 말풍선/뱃지로 강조
  - 예: "훅 강도: 강력" 뱃지, "시각 변화량 51" 등

### 현재 대비 변경점
- "떡상한 커머스 영상 분석하세요" → 위 헤드라인으로 교체
- "지금 시작하기" 버튼 → URL 입력창으로 교체
- 우측에 실제 스크린샷 추가

## 5. 섹션3: 작동 방식 (4단계)

### 레이아웃
가로 4단계 (현재 Spike→Hack→Script→Expand 위치를 재활용)
각 단계에 번호 + 아이콘 + 제목 + 설명 + 스크린샷 썸네일

### 카피

**섹션 타이틀:**
```
WORKFLOW
이렇게 작동합니다
```

**단계 1:**
```
번호: 01
제목: 분석
설명: 영상 URL을 넣으면 구조를 컷 단위로 해부합니다.
      훅, 소구, 리듬, 설득 흐름을 자동으로 분석합니다.
스크린샷: 분석 리포트 요약 탭 캡처
```

**단계 2:**
```
번호: 02
제목: 비교
설명: 잘 되는 영상과 내 영상을 나란히 놓고,
      구조적으로 뭐가 다른지 보여줍니다.
스크린샷: 비교 화면 캡처 (2개 영상 나란히 + 수치 비교)
※ 비교 화면이 아직 없으면: 분석 리포트 2개를 나란히 배치한 목업
```

**단계 3:**
```
번호: 03
제목: 코칭
설명: 어디를 어떻게 고쳐야 하는지,
      데이터 근거와 함께 알려줍니다.
스크린샷: 코칭 탭 캡처 (잘된 점 + 개선 포인트 + 추천 구조)
```

**단계 4:**
```
번호: 04
제목: 제작
설명: 분석된 구조를 내 제품에 맞는
      대본으로 만들어줍니다.
스크린샷: 제작가이드 리포트 캡처 (구조 개선 + 대본 초안)
```

### 현재 대비 변경점
- Spike → 분석, Hack → 비교, Script → 코칭, Expand → 제작
- 영어 네이밍 제거, 한국어로 통일
- 각 단계에 실제 스크린샷 썸네일 추가

## 6. 섹션4: 실제 분석 데모

### 목적
마케터가 "와 이런 걸 해주는구나"를 텍스트가 아니라 실제 결과물로 느끼는 구간.
**이 섹션이 전환율을 가장 크게 좌우함.**

### 레이아웃
넓은 영역에 실제 분석 결과를 인터랙티브하게 또는 슬라이드로 표시.

### 구현 방식 (택 1)

**방식 A — 스크린샷 슬라이드 (쉬움)**
- 실제 분석 결과 스크린샷 4~5장을 캐러셀(슬라이드)로
- 슬라이드 내용:
  1. 요약 탭 전체 화면
  2. 통합 타임라인 (컬러바 + 변화량 곡선)
  3. 코칭 (잘된 점 + 개선 포인트)
  4. 제작가이드 리포트
  5. 대본 생성 결과
- 하단에: "이 영상의 전체 분석 결과 보기 →" 링크 (실제 리포트 페이지로)

**방식 B — 인터랙티브 데모 (고급)**
- 실제 분석 결과 데이터를 정적으로 렌더링
- 4개 탭을 클릭하면 해당 탭 내용이 보임 (실제 서비스와 동일한 UI)
- 하단에: "내 영상도 분석해보기 →" CTA

### 카피

**섹션 타이틀:**
```
LIVE DEMO
실제 분석 결과를 확인해보세요
```

**서브:**
```
32초 딸기 푸딩 숏폼을 와카랩이 분석한 결과입니다
```

### 스크린샷 목록 (캡처 필요)
1. `demo_summary.png` — 요약 탭 (영상 요약 + 메타 + 핵심 코칭)
2. `demo_timeline.png` — 통합 타임라인 (컬러바 + 곡선 + 스크립트)
3. `demo_coaching.png` — 코칭 탭 (잘된 점 + 개선 + 추천 구조)
4. `demo_guide.png` — 제작가이드 (구조 개선 + 레시피 리팩토링)
5. `demo_script.png` — 대본 생성 결과

**스크린샷 준비:** 와카랩에서 딸기 푸딩(또는 다른 예시) 분석 결과 각 탭을 풀스크린으로 캡처.
민감한 정보 없는 공개 영상 사용.

## 8. 섹션6: 사회적 증거

### 목적
"다른 사람도 쓰고 있다"는 신뢰감.
베타 초기라 레퍼런스가 적으므로 가볍게.

### 레이아웃
중앙 정렬, 수치 + 로고 또는 인용

### 구현 (택 1 또는 조합)

**방식 A — 실시간 카운터:**
```
지금까지 분석된 영상
[XXX] 건
```
실제 results 테이블 COUNT를 주기적으로 업데이트.
베타 오픈 후 숫자가 올라가면 사회적 증거가 됨.

**방식 B — 인용:**
```
"잘되는 영상과 비교해서 뭐가 다른지 바로 보이니까,
클라이언트 제안할 때 근거가 생깁니다"
— 마케팅 에이전시 AE
```
실제 베타 유저 피드백이 생기면 업데이트.
초기에는 이종권 CBO 또는 내부 테스터 인용.

**방식 C — 로고 (가능하면):**
- GS리테일 로고 (본계약 체결 사실이 공개 가능하면)
- "활용 기업" 섹션
- 불가능하면 이 섹션은 베타 이후 추가

### 추천
베타 오픈 시에는 **방식 A(카운터) + 방식 B(인용 1개)**만.
로고는 허가 받은 후 추가.

## 13. 반응형 (모바일)

- 데모: 캐러셀 유지 (스와이프)
- 기능: 3+2 그리드 → 세로 스택

- 하단 CTA: URL 입력창 전체 너비

## 14. 인터랙션 + 이벤트 추적

### URL 입력 플로우
```
히어로 또는 하단 CTA에서 URL 입력
  ↓
URL 유효성 검사 (틱톡/인스타/유튜브 URL인지)
  ├── 유효하지 않음 → "틱톡, 인스타 릴스, 유튜브 숏츠 URL만 지원합니다" 인라인 에러
  └── 유효함 → URL을 세션/로컬에 저장 → 가입 페이지로 이동
       ↓
     가입 완료 → 저장된 URL로 즉시 분석 시작 → 분석 결과 페이지로
     이미 로그인 → 즉시 분석 시작
```

### 이벤트 추적 (user_activity_logs 또는 별도 analytics)
| 이벤트 | 위치 | metadata |
|--------|------|----------|
| `landing_url_input` | 히어로/하단 CTA | {url, source: "hero" or "bottom"} |
| `landing_url_invalid` | 히어로/하단 CTA | {url, error_reason} |
| `landing_cta_click` | 모든 CTA 버튼 | {button_location} |
| `landing_demo_view` | 섹션4 데모 | {slide_index} |
| `landing_pricing_view` | 섹션7 요금제 | {} |
| `pro_interest_click` | 요금제 Pro 카드 | {} |
| `landing_section_scroll` | 전체 | {section_name, time_on_section} |

### 스크롤 기반 애니메이션
- 각 섹션 진입 시 fade-in + slide-up (가벼운 트랜지션)
- 작동 방식 4단계: 순차적으로 나타남 (stagger delay)
- 데모 섹션: 스크롤 도달 시 첫 슬라이드 자동 표시
- 카운터(사회적 증거): 뷰포트 진입 시 0에서 카운트업 애니메이션
