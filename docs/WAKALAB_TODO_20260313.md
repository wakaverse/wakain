## 🔴 P0 — 분석 품질 다듬기 (베타 오픈 전 필수)

### T1. 지표 이름 변경 + 기준선 제공

**⚠️ 용어 변경 (전체 적용):**
- ~~에너지~~ ~~어텐션~~ → **시각 변화량 (Visual Dynamics)**
- 이 지표는 시청자 행동 데이터가 아니라 **영상 자체의 물리적 변화량**을 측정한 것임
- P3의 edge_diff(엣지 변화) + color_diff(컬러 변화) 조합
- "어텐션"이라고 하면 마케터가 "시청자 집중도"로 오해함 → 반드시 이름 변경
- 코드 내 변수명/필드명도 가능하면 `attention_curve` → `dynamics_curve` 등으로 점진 변경
- 프론트 표시: "첫 3초 에너지 46" → "첫 3초 시각 변화량: 46" 또는 "Visual Dynamics: 46"

**문제:** "첫 3초 시각 변화량 46"이라는 숫자를 보고 마케터가 높은 건지 낮은 건지 판단 불가.
이 지표는 크랩스 독자 지표라서 업계 기준이 없음.

**해결 (3가지 모두 적용):**
1. **기준선 표시** — 분석 완료된 전체 영상의 평균값을 계산하여 "전체 평균: XX" 또는 추후 "식품 카테고리 평균: XX" 표시
2. **구간 등급 뱃지** — 0~30 낮음 / 31~60 보통 / 61~100 높음 (또는 데이터 기반 분위수로 조정)
3. **타임라인 그래프에 평균선** — dynamics_curve 그래프에 점선으로 평균 기준선 1개 추가

**데이터 소스:** `visual.attention_curve` (→ 추후 `visual.dynamics_curve`), `P5.attention_points`

### T2. 훅 분석 "강력" 뱃지 — 스코어링 로직 투명화
**문제:** 훅 분석에 "강력" 뱃지가 표시되지만 근거가 불명확. 사용자가 왜 강력한지 납득 불가.

**해결:**
- 훅 강도 뱃지 옆에 1줄 근거 표시 (P9.hook_reason 활용) — 예: "강력: 가격 혜택 + 신품종 호기심 동시 자극"
- 또는 훅 강도도 수치화하여 "훅 점수 78/100" 같은 형태로 전환 검토
- ~~체크리스트와의 모순 문제~~ → T3에서 체크리스트 자체를 제거하므로 해결됨

**데이터 소스:** `engagement.retention.hook_strength`, `engagement.retention.hook_reason`

### T3. 체크리스트 제거 + 섹션 순서 재배치
**결정:** 체크리스트(✅/❌ 목록)를 사용자 화면에서 제거.
- 체크리스트는 "너 이거 못했어"라는 인상을 줌 — 내부 평가 도구이지 사용자 콘텐츠가 아님
- 체크리스트에서 유용한 정보는 코칭 섹션(improvements[])에서 자연스럽게 전달
- P13에서 체크리스트 데이터는 계속 생성하되, 프론트에서 비노출 처리
- (추후 B2B 엔터프라이즈 버전에서 상세 리포트 옵션으로 부활 가능)

**섹션 순서 재배치 (줌아웃 → 줌인 원칙):**
1. **영상 요약** — "이 영상이 뭔데?" (한 줄 요약, 메타 정보)
2. **콘텐츠 포지셔닝** — "어떤 전략으로 만들었는데?" (차별 각도, 대비 구조)
3. **훅 분석** — "첫인상은 어땠는데?" (훅 강도, 3초 시각 변화량)
4. **통합 타임라인** — "시간 순서로 뭐가 일어나는데?" (T4에서 상세)
5. **제품 소구** — "뭘 말하고 있는데?" (소구 유형별 분류)
6. **코칭** — "그래서 뭘 고치라는데?" (강점 + 개선)

**전체 섹션 접기/펼치기 적용:**
- 영상 요약만 기본 펼침 — 나머지 섹션은 기본 접힘(collapsed) 상태
- 각 섹션 헤더에 핵심 1줄 요약을 표시하여 접힌 상태에서도 내용 파악 가능
- 코칭 섹션의 핵심 1줄은 영상 요약 카드에도 추가
- 이렇게 하면 첫 화면이 곧 요약 대시보드 역할 → 별도 대시보드 불필요

### T4. 씬 분석 → 통합 타임라인 뷰로 전환
**문제:** 현재 씬 분석이 카드 형태로 나열되어 시계열 정보가 끊김.
설득 흐름 컬러바, 시각 변화량 곡선, 씬 분석이 각각 따로 존재하는데 셋 다 시계열 데이터.

**해결: 하나의 통합 타임라인으로 병합**

가로축 = 시간. 세로로 레이어를 쌓는 구조:
- **1층: 설득 흐름 컬러바** — 문제/공감, 차별점, 장점소개 등 (현재 이미 있음)
- **2층: 시각 변화량 곡선** — 어느 구간이 시각적으로 강한지 약한지 (현재 별도 타임라인 그래프)
- **3층: 스크립트** — 해당 시간에 뭐라고 말하는지 (현재 씬 분석 카드에 분산)
- **4층: 소구 태그** — 경험전환, 기능적, 감성적 등 (현재 제품 소구 분석에 분산)

**인터랙션:**
- 세로로 보면 "이 시점에 무슨 일이 일어나고 있나" 한눈에 파악
- 가로로 보면 "이 영상이 어떤 흐름으로 설득하나" 파악
- 특정 구간 클릭 시 해당 시점의 상세 정보 펼침 (썸네일, shot type, visual forms, 코칭 제안)
- 이탈 위험 구간(risk_zones)은 배경색으로 표시

**기존 UI 제거/통합:**
- 기존 "설득 흐름" 카드 → 통합 타임라인 1층으로 흡수
- 기존 "타임라인" 그래프 → 통합 타임라인 2층으로 흡수
- 기존 "씬 분석" 카드 목록 → 통합 타임라인 클릭 상세로 흡수

**핵심 원칙:** 의미가 주인공, 썸네일은 부속품. 마케터는 예쁜 사진이 아니라 "이 구간에서 왜 이 기법을 썼고 효과가 있었는지"를 알고 싶음.

**데이터 소스:** `script.blocks`, `visual.scenes`, `visual.dynamics_curve`, `engagement.risk_zones`, `product.claims`

### T11. P4 CLASSIFY 안정화 (베타 전 필수)
**문제:** 최근 분석에서 TypeError 발생, 재시도 2회 후에도 실패. 
Flash Lite가 8장 배치 프레임 분류 시 간헐적으로 JSON 응답 구조가 깨지는 것으로 추정.

**해결:**
- 응답 파싱에 방어 로직 추가 (try-catch + 필드별 기본값 폴백)
- 배치 전체 재시도가 아니라, 실패한 프레임만 개별 재시도
- 3회 실패 시 해당 프레임은 기본값으로 채우고 진행 (분석 전체가 멈추지 않도록)
- 실패 로그를 pipeline_logs에 기록 (T14 모니터링과 연계)

**난이도:** 낮음
**긴급도:** 높음 — 베타에서 사용자 영상 분석이 실패하면 첫인상 치명적

## 🟠 P1.7 — 인프라 안정화 (베타 오픈 전 필수)

### T15. Gemini API 장애 대응
**문제:** P2, P4, P7, P8, P9, P10, P13 — 7개 Phase가 Gemini 의존. Gemini 장애 = 전체 중단.

**해결 (최소한):**
- 모든 Gemini 호출에 자동 재시도 3회 (exponential backoff: 2초, 4초, 8초)
- 재시도 후에도 실패 시 사용자에게 "분석 중 오류 발생, 잠시 후 다시 시도해주세요" 안내
- 실패 건은 jobs 테이블에 상태 기록 → 수동 재실행 가능하게
- (장기) Gemini 외 fallback 모델 검토 (Claude API 등)

## 🔵 P3 — 온톨로지 브릿지 (기초 작업)

### T9. 시간축 겹침 기반 엔티티 관계 쿼리
**현재 상태:** RecipeJSON에 이미 시간축을 공유하는 데이터가 존재
- scenes[].time_range
- script.blocks[].time_range  
- product.claims[].source (타임스탬프)
- visual.attention_curve (초 단위)
- engagement.risk_zones[].time_range

**구현:**
- time_range overlap 쿼리 함수 생성
- 입력: 시간 구간 (start, end)
- 출력: 해당 구간에 겹치는 모든 엔티티 (발화, 씬, 소구, 블록, 컷, 리스크존)
- 별도 온톨로지 엔진 불필요 — SQL/JSONB 쿼리로 충분
- 6개 엔티티(Utterance ↔ Scene ↔ Appeal ↔ Element ↔ Cut ↔ Product) 관계를 시간 겹침으로 자동 생성
