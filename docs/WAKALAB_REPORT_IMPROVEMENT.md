# WakaLab 분석 리포트 개선 태스크

> 2026-03-14
> Claude Code 실행용. 분석 리포트 UI/UX + 코칭 품질 개선.

---

## 1. 리포트 → 4탭 구조 전환

### 현재
- 모든 섹션이 세로 스크롤로 나열 (접기/펼치기 적용 상태)
- 정보 과부하 — 마케터가 전부 볼 시간 없음
- 어떤 정보에 가치를 느끼는지 파악 불가

### 변경
스크롤 나열을 제거하고, 4개 탭으로 분리.

| 탭 | 이름 | 내용 | 역할 |
|----|------|------|------|
| 1 | **요약** | 한 줄 요약 + 메타 정보(길이/플랫폼/컷수/노출률) + 핵심 코칭 1줄 | 3초 안에 결과 파악. 기본 탭. |
| 2 | **타임라인** | 통합 타임라인 (설득 흐름 컬러바 + 시각 변화량 곡선 + 스크립트 + 소구 태그 + 썸네일) | 시간 순서로 영상 파악 |
| 3 | **코칭** | 잘된 점 + 개선 포인트 + 추천 구조(현재 vs 제안) + 제작가이드 버튼 | "뭘 고치라고?" |
| 4 | **구조 분석** | 콘텐츠 포지셔닝 + 훅 분석 + 제품 소구 분석 | 깊이 파고 싶은 사람용 |

### 기존 섹션 → 탭 매핑

| 기존 섹션 | 이동할 탭 |
|----------|---------|
| 영상 요약 | 탭1: 요약 |
| 핵심 코칭 (요약 카드 내) | 탭1: 요약 |
| 통합 타임라인 | 탭2: 타임라인 |
| 코칭 — 잘된 점 | 탭3: 코칭 |
| 코칭 — 이렇게 하면 더 좋아요 | 탭3: 코칭 |
| 코칭 — 추천 구조 | 탭3: 코칭 |
| 코칭 — 구간별 코칭 (Hook/Body/CTA) | 탭3: 코칭 |
| 제작가이드 버튼 | 탭3: 코칭 (하단) |
| 콘텐츠 포지셔닝 | 탭4: 구조 분석 |
| 훅 분석 | 탭4: 구조 분석 |
| 제품 소구 분석 | 탭4: 구조 분석 |

### 제거 항목
- 체크리스트 — 프론트 비노출 (P13에서 데이터는 계속 생성, 프론트에서만 제거)

### 탭 클릭 로그
- 탭 전환 시 이벤트 기록
- 형식: `{ action: "tab_click", metadata: { tab_name: "coaching" }, result_id: "xxx" }`
- 저장: user_activity_logs 테이블 (테이블 없으면 생성)
- 목적: 베타 후 "마케터들은 어떤 탭을 가장 많이 보는가" 데이터 확보

## 2. 코칭 품질 — P13 프롬프트 개선

### 현재 문제
코칭이 일반론으로 가득함. 예시:
- ❌ "숏폼에서는 초반에 제품을 빨리 노출하는 것이 중요합니다"
- ❌ "시각적 변화가 빠르면 이탈을 줄일 수 있습니다"
- ❌ "CTA에 긴급성을 더하면 좋습니다"
→ 마케터가 이미 아는 교과서. 돈 낼 가치 없음.

### 개선 원칙

**금지:**
- "~하는 것이 중요합니다" 형태의 일반론
- "~하면 좋습니다" 형태의 뻔한 제안
- 이 영상 데이터와 무관한 일반 숏폼 팁

**필수:**
- 모든 코칭에 이 영상의 정량 데이터 근거 포함
- 구간 지정 (몇 초~몇 초)
- 비교 기준 제시 (카테고리 평균 대비, 또는 영상 내 다른 구간 대비)

### 개선 예시

**잘된 점:**
- ❌ 기존: "전체 영상에서 제품 노출 비중이 95%로 매우 높았습니다."
- ✅ 개선: "제품 노출률 95%. 식품 카테고리 평균 72% 대비 23%p 높음. 0초부터 제품이 등장해 인지 지연 없음."

**개선 포인트:**
- ❌ 기존: "영상에 '이런 고민 있죠?'와 같은 시청자의 실제 고민을 시각적으로 보여주는 장면이 부족합니다. ... 비싼 디저트 값에 놀라거나, 아이 간식으로 무엇을 줄지 고민하는 엄마의 모습을 짧게라도 연출하여 공감대를 강화해 보세요."
- ✅ 개선: "0:02~0:05 문제/공감 구간: 텍스트 멘트만 존재, 시각적 공감 장면 없음. 이 구간 시각 변화량 42 (영상 평균 55 대비 낮음). 제안: 고민 상황 시각화 1~2초 삽입 시 변화량 + 공감 동시 확보 가능."

### P13 프롬프트에 추가할 지시문 (예시)

```
코칭 작성 규칙:
1. 일반적인 숏폼 제작 조언은 절대 하지 마세요. 사용자는 이미 알고 있습니다.
2. 이 영상의 분석 데이터에서만 발견되는 구체적 사실만 코칭하세요.
3. 모든 코칭 항목에 반드시 포함:
   - 해당 구간 시간 (예: 0:05~0:12)
   - 정량 수치 (예: 시각 변화량 38, 경험 소구 비율 58%)
   - 비교 기준 (예: 영상 평균 대비, 카테고리 평균 대비)
4. "~하는 것이 중요합니다", "~하면 좋습니다" 형태의 문장은 사용하지 마세요.
5. 제안은 구체적 행동으로: "공감 장면 추가" (O), "공감대를 강화해 보세요" (X)
6. 잘된 점은 3개 이내, 개선 포인트는 3개 이내. 양보다 질.
```

### 코칭 텍스트 양 목표
- 현재 대비 절반 이하
- 잘된 점: 최대 3개, 각 2~3줄
- 개선 포인트: 최대 3개, 각 2~3줄 (구간 + 수치 + 제안)
- 구간별 코칭 (Hook/Body/CTA): 각 2~3줄

## 3. 통합 타임라인 개선

### 3-1. 썸네일 뷰 추가

**현재:** 컬러바 + 변화량 곡선 + 스크립트 텍스트(잘림) + 소구 태그
**추가:** 씬별 대표 썸네일

**구현:**
- 컬러바와 변화량 곡선 사이, 또는 컬러바 바로 아래에 썸네일 행 추가
- 각 씬의 대표 프레임 (P3에서 추출된 frames[] 중 씬 시작 프레임)
- 타임라인 비율에 맞게 배치 (씬 길이에 비례한 너비)
- 텍스트 뷰 / 썸네일 뷰 토글 버튼, 또는 둘 다 표시

**데이터 소스:** P3 frames[] (이미 존재), recipe_json.scenes[].time_range

### 3-2. 스크립트 텍스트 잘림 개선

**현재:** 가로 공간에 블록 텍스트가 우겨넣어져서 "체...", "체..." 로 잘림
**현재 이미 있는 것:** 마우스 호버 시 툴팁으로 전체 텍스트 표시, 클릭 시 해당 지점 영상 재생

**개선 방향:**
- 텍스트 라벨에 블록 유형만 표시 (예: "가치", "신뢰", "경험", "기능")
  → 현재 "이거 모르고 디저트..." 대신 소구 태그 아이콘 + 유형명만
- 상세 스크립트는 툴팁(이미 있음) + 클릭 시 하단 패널에 표시
- 또는: 타임라인 아래에 "현재 선택된 구간" 상세 패널 (스크립트 + 소구 + 썸네일)

## 4. 제품 소구 분석 개선

### 현재 문제
가치 2개, 신뢰 1개, 경험 7개, 기능 2개가 리스트로 쭉 나열.
정보는 많은데 "그래서 뭐?" 가 없음.

### 변경

**상단: 소구 비율 차트 + 한 줄 해석**
- 원형 차트 또는 가로 막대 차트: 유형별 비율 시각화
- 한 줄 해석 예시: "경험 소구 58% (7건) — 감각적 묘사 중심. 가치/신뢰 소구는 상대적으로 부족."
- (추후 벤치마크 연동 시: "식품 카테고리 평균 경험 소구 40% 대비 18%p 높음")

**하단: 개별 소구 상세 (펼치기)**
- 기본 접힘 상태
- 소구 유형별 그룹핑 (경험: 7건, 가치: 2건, 신뢰: 1건, 기능: 2건)
- 펼치면 현재와 같은 개별 소구 리스트

**데이터 소스:** product.claims (이미 존재), claim_type별 집계

## 실행 순서

| 순서 | 태스크 | 난이도 | 이유 |
|------|--------|--------|------|
| 1 | T5. 용어 변경 | 낮음 | 텍스트 치환, 바로 가능 |
| 2 | T1. 4탭 구조 전환 | 중간 | 리포트 전체 레이아웃 변경. 가장 체감 큰 변화. |
| 3 | T4. 소구 분석 개선 | 낮음 | 차트 + 접기. 탭4(구조분석) 안에서 개선. |
| 4 | T3. 타임라인 개선 | 중간 | 썸네일 뷰 + 텍스트 잘림. 탭2(타임라인) 품질. |
| 5 | T2. P13 프롬프트 개선 | 중간 | 코칭 품질. 프롬프트 수정 후 재분석 필요. |

T1(탭 구조)을 먼저 해야 나머지가 각 탭 안에서 작업 가능.
T2(P13)는 프롬프트 수정이라 코드 변경은 적지만 품질 검증에 시간 필요.
