# T16: 소셜미디어 API 대안 조사 리포트

> 작성일: 2026-03-14
> 목적: RapidAPI 의존도 줄이기 + 플랫폼 정책 변경 대비

## 배경

WakaIn 레이더 기능은 현재 RapidAPI 마켓플레이스의 Instagram/TikTok 스크래핑 API에 의존 중.
이 방식은 다음 위험이 있음:
- RapidAPI 내 개별 프로바이더가 서비스 중단할 수 있음
- Instagram/TikTok의 스크래핑 차단 정책 강화 시 일괄 중단
- 프로바이더별 응답 포맷 불일치, 품질 편차

## 서비스별 상세 분석

### 1. Instagram Graph API (공식)

**장점:**
- Meta 공식 API로 법적 리스크 없음
- 안정적이고 문서화 잘 되어 있음
- 무료

**단점:**
- 자기 계정 데이터만 접근 가능 → WakaIn 레이더의 "경쟁사/트렌드 분석" 기능에 부적합
- 비즈니스/크리에이터 계정 연동 필요
- 앱 심사 과정 필요

**WakaIn 적합도: 낮음**
경쟁사 계정 분석이 핵심인 레이더 기능에는 사용 불가. 자사 계정 인사이트 보조 용도로만 활용 가능.

---

### 2. TikTok Research API

**장점:**
- TikTok 공식 API로 법적 리스크 없음
- 공개 영상 검색/메타데이터 접근 가능
- 무료

**단점:**
- 승인 절차가 까다로움 (학술/비즈니스 목적 증명 필요)
- 제공 데이터 범위가 제한적 (댓글, 팔로워 상세 등 미제공)
- 상업적 사용에 대한 제한 가능

**WakaIn 적합도: 중**
승인만 받으면 TikTok 트렌드 분석에 활용 가능하나, 승인 불확실성이 리스크.

---

### 3. Apify

**장점:**
- Instagram Scraper, TikTok Scraper 등 검증된 Actor 다수 존재
- 자체 프록시 풀로 차단 대응 능력 높음
- Actor가 차단되면 Apify 팀이 자동 업데이트
- $49/mo Starter 플랜으로 충분한 볼륨
- API + 웹훅 지원, 파이프라인 연동 용이

**단점:**
- 스크래핑 기반이므로 플랫폼 ToS 위반 가능성
- Actor 업데이트 지연 시 일시적 데이터 공백 가능
- 대량 수집 시 비용 증가

**WakaIn 적합도: 높음**
현재 RapidAPI 방식과 동일한 데이터를 더 안정적으로 수집 가능. 가격 대비 성능 우수.

---

### 4. Bright Data (구 Luminati)

**장점:**
- 엔터프라이즈급 안정성
- 소셜미디어 전용 데이터셋 제공 (사전 수집된 구조화 데이터)
- 가장 넓은 프록시 풀 (72M+ IP)
- Web Scraper IDE로 커스텀 수집 가능

**단점:**
- 월 $500+ 비용은 현재 WakaIn 규모에 과도
- 초기 설정 복잡도 높음
- 소규모 스타트업에는 오버스펙

**WakaIn 적합도: 낮음 (현재 단계)**
서비스 규모가 커지면 고려할 수 있으나, 현재는 비용 대비 효용이 낮음.

---

### 5. 현재 RapidAPI 유지 + graceful degradation

**구현 방안:**
- API 호출 실패 시 마지막 성공 데이터를 캐시에서 반환
- 3회 retry + exponential backoff
- 연속 실패 시 Slack/알림 발송
- 프로바이더 2~3개 등록 후 failover 구성

**장점:**
- 추가 비용/마이그레이션 없음
- 기존 코드 최소 변경

**단점:**
- 근본적 해결이 아님
- 프로바이더 전체 중단 시 대응 불가

**WakaIn 적합도: 중 (단기 대응)**

## 추천 전략

### 1순위: Apify로 마이그레이션

**이유:**
1. **데이터 커버리지**: Instagram + TikTok 모두 지원, 경쟁사 분석 가능
2. **안정성**: 자체 프록시 풀 + Actor 자동 업데이트로 RapidAPI 대비 안정적
3. **비용 효율**: $49/mo Starter로 현재 사용량 충분히 커버
4. **통합 용이**: REST API + 웹훅 제공, 기존 파이프라인에 연동 쉬움
5. **전환 비용**: Actor 호출 방식이 RapidAPI와 유사해 마이그레이션 부담 적음

### 병행 전략

| 단계 | 액션 | 시기 |
|------|------|------|
| **즉시** | 현행 RapidAPI에 graceful degradation 적용 (캐싱 + retry + 알림) | R19 내 |
| **단기** | Apify Actor 검증 → 파일럿 전환 (Instagram 먼저) | R20 |
| **중기** | TikTok Research API 신청 → 승인 시 TikTok 공식 API 병행 | R20-R21 |
| **장기** | 트래픽 증가 시 Bright Data 검토 | 서비스 스케일업 시 |
