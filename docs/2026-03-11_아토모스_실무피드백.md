# 아토모스 실무 피드백 (2026-03-11)

**출처:** 아토모스 실무 담당자 — 류주희 팀장, 류민지 팀장
**정리 기준:** WakaLab V2 현재 상태 대비 Gap 분석

---

## 1. 소구점 2계층 구조 🔴 최우선

**현업 핵심:** "베타카로틴이 뭔지가 중요한 게 아니라, 그게 너한테 어떤 경험을 줄 수 있는지를 풀어서 고객이 이것만 보고도 효과를 느낄 수 있어야 한다." (류주희)

**현재:** claims = 제품 속성만 추출
**필요:** 제품 속성 → 소비자 경험 번역 과정 자체가 소구점

### 2계층 구조
- 1계층: 제품 속성 (현재와 동일) — 예: 베타카로틴 함유, 저칼로리, 유기농
- 2계층: 설득 번역 — 해당 속성이 영상 내에서 어떤 설득 전략으로 번역되었는가

### 설득 전략 분류 (아토모스 미팅에서 추출)
- **경험 전환형**: 스펙 → 소비자 일상 경험 ("아침마다 거울 보는 게 즐거워진다")
- **손실 회피형**: 안 사면 손해 프레이밍 ("이번에 못 사면 내년까지 기다려야 합니다")
- **정보 선점형**: 정보 제공으로 신뢰 구축 후 전환 ("가짜 대저 토마토 구별법")
- **사회적 증거형**: 타인 반응/리뷰로 설득 ("시어머니가 물어보셨다")
- **가격 앵커링형**: 비교 가격으로 저렴함 부각 ("가짜보다 저렴하게")

**추가 인사이트:** 경쟁사 분석 시 다른 카테고리에서 설득 방식을 가져옴 → 소구점은 카테고리에 종속되지 않음

## 2. 후킹 구간 0~8초 요소별 통계 🔴 높음

**현업 핵심:** "앞단 3초, 길게는 8초 안에 어떤 요소를 표현했는지가 통계적으로 나오면 좋겠다" (류민지)

### 필요한 요소 분석
- 소구점 유형 (가격/경험/정보/감정 등)
- 텍스트 배너 유무 및 내용
- 인물 등장 유무
- 제품 직접 노출 유무
- 효과음/BGM 변화 유무
- 화면 전환 빈도

### 방향: "후킹 스캔" 독립 모듈
- 전체 3~4분 대비 30초 이내
- 대량 비교 가능 (카테고리 50개 영상 스캔)
- A/B 테스트 설계에 활용
- "후킹 카드" 시각화 — 요소 조합을 한눈에

---

## 3. 성과 기여 요소 분리 (Attribution) 🟢 중장기

**현업 핵심:** "요소가 많이 들어가서 잘 팔린 건지, 영상 소스 자체로 잘 된 건지 구분이 됐으면 좋겠다. 배너를 안 써도 되면 아예 안 써버리겠다." (류민지)

### 접근
- **단기:** 요소 밀도 분석 (배너 수, 자막 길이, 효과음 횟수, 컷 전환 빈도, 인물/제품 노출 비율)
- **중기:** ATMS 성과 데이터(ROAS) 연결 → 상위 30% vs 하위 30% 이진 분류
- **장기:** 회귀분석 → "배너 유무가 전환율에 미치는 영향: +3.2%"

전제: 아토모스 측 성과 데이터 라벨링 선행 필요

---

## 4. 레이아웃 및 시선 흐름 분석 🟢 후순위

**현업 핵심:** 배너 위치(상단/중앙/하단), 자막 배치별 분석 요청 (류민지)

### 방향
- 화면 9분할(3×3 그리드) → 프레임별 요소 위치 추적
- Eye Tracking은 과도 → "화면 영역별 요소 맵핑"으로 대체
- 레이아웃 패턴 자동 분류 ("상단 텍스트 + 중앙 제품", "하단 자막 + 전면 인물")
- **우선순위 낮음** — 소구점/후킹 이후

---

## 5. 분석 모드 프리셋 🟡 중

**방향:** 항목 개별 선택 ❌ → 용도별 프리셋 ✅ (마케터는 14개 항목 뭘 골라야 하는지 모름)

### 3단계 모드
- **퀵 스캔 (30초):** 후킹 구간 + 소구점 + 컷/리듬. 경쟁사 대량 비교용.
- **표준 분석 (2분):** 퀵 + 설득 구조 + 제품/인물 노출도 + 레이아웃. 일반 리뷰용.
- **풀 분석 (4분):** 현재 14단계 전체. 심층 연구용.

**캐싱:** 퀵→풀 재분석 시 이미 분석된 부분은 건너뛰기

---

## 6. 경쟁사 비교 분석 / 컬렉션 🟡 중

**현업 핵심:** 경쟁사 분석을 "노가다로 하나씩" 하고 있음 (류민지)

### "컬렉션" 기능 제안
- **카테고리별 자동 그룹핑** → 카테고리 내 공통 패턴 추출
- **스파이크 탐지** → 조회수 급등 영상 자동 플래그 + 공통 요소 분석
- **비교 분석** → 두 영상/그룹 나란히 놓고 차이 시각화
- ATMS 연동 시 수백 개 소재 자동 분석 가능

## 7. 분석 → 생성 연결 🔴 최우선 (양 팀장 공통 최강 요청)

**류주희:** "분석만으로는 루킹용밖에 안 된다. 와카쇼츠랑 연동이 돼야 한다."
**류민지:** "프리셋을 만들어놓으라는 것. 레퍼런스 영상 넣으면 그 느낌을 분석해서 내 상품에 적용."
**류주희:** "신입이 와도 70%의 수준은 바로 가능한 상태."

### 시나리오
1. 레퍼런스 영상 → WakaLab 분석
2. 소구점/후킹/리듬/레이아웃 추출
3. "이 스타일로 우리 상품 영상 만들어줘"
4. 내 상품 상세페이지 입력 → 소구점 자동 추출
5. 레퍼런스 스타일 + 내 소구점 결합 → 대본 생성
6. WakaShorts로 영상 자동 생성

### "스타일 프리셋" 시스템
저장 항목:
- 소구 전략 유형
- 후킹 패턴
- 영상 리듬 (컷 평균 길이, 전환 빈도, 템포 곡선)
- 구성 비율 (인물/제품 노출)
- 대본 구조 (도입-본문-CTA 비율, 설득 순서)

→ "아토모스 식품 스타일 A" 같이 이름 붙여 팀 공유 가능

## 8. 상세페이지 크롤링 → 소구점 추출 🔴 높음

**류주희:** "상세페이지 링크 넣었을 때 키워드들이 스크래핑돼서 분석되면 좋겠다" → "너무 써보고 싶다"

- GS리테일에 이미 구현됨 → 아토모스 맥락으로 확장
- 크롤링 지원 순서: 자사몰 → 스마트스토어 → 쿠팡 (쿠팡은 버튼 2번 구조라 어려움)
- 2계층 소구점 구조 적용: 속성 추출 → 설득 전략별 번역 → 후보 여러 개 → 마케터 선택

---

## 9. 성과 라벨링 체계 🟡 외부 의존

**류주희:** "안 됐다는 거에 매출 기준이 애매하다"

### 단계별 접근
1. **즉시:** ROAS 기준 상위 30%(성공) / 하위 30%(실패) 이진 분류
2. **ATMS 연동 후:** 자동 라벨링 (영상 등록 + 성과 자동 태깅)
3. **데이터 축적 후:** 성과 예측 ("이 영상 예상 등급: B+")

**핵심:** 라벨링 없이는 "이런 구조다(기술)" 에서 "이런 구조가 잘 먹힌다(처방)"로 못 감

---

## 10. 자동화 불가 영역 — 리소스 투입 안 함

**양 팀장 공통:** "소구가 훨씬 중요하다", "효과는 시간 남으면 하라고 한다"

자동화 불가:
- 효과음 종류 인식 (띵동, 팡 등)
- 음성 감정/톤 파악
- 모션그래픽, 불투명도 등 고급 연출
- 음악 박자에 맞춘 편집 타이밍
- 순간적 화면 효과 (번쩍, 줌 등)

**장기 방향:** WakaShorts 하이브리드 편집 모드 (80% 자동 + 20% 수동)
