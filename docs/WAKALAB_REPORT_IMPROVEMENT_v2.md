## 개선 사항 (우선순위순)

### 1. 타겟 오디언스 영어 → 한국어

**현재:**
```
This video targets consumers interested in high-quality, fresh citrus fruits, 
particularly those looking for premium gift options or seeking information 
about the origin and quality of their produce.
```

**문제:** 한국 마케터 대상 서비스에서 영어 문구가 나옴. 신뢰도 하락.

**해결:** P2(SCAN) 프롬프트에서 타겟 오디언스 출력을 한국어로 지정.
```
예시: "고품질 신선 감귤에 관심이 있는 소비자, 특히 프리미엄 선물 세트를 찾거나 
원산지와 품질 정보를 중요시하는 사람들을 대상으로 합니다."
```

**수정 위치:** P2 SCAN 프롬프트의 target_audience 출력 언어 지정.

### 2. 핵심 코칭 — 요약 반복 제거

**현재:**
요약 탭의 "영상 요약" 텍스트와 "핵심 코칭" 텍스트가 거의 동일.
둘 다 "제주 프리미엄 레드향을 판매하는 영상입니다. 겉만 번지르르한 과일이 아닐까 하는 시청자의 잠재적 의구심을..."

**문제:** 핵심 코칭이 요약의 복붙. 코칭의 가치 없음.

**해결:** 핵심 코칭은 **"이 영상에서 가장 고쳐야 할 한 가지"**만.

**P13 프롬프트에 핵심 코칭 규칙 추가:**
```
핵심 코칭(summary)은 영상 요약을 반복하지 마세요.
이 영상에서 가장 개선 효과가 클 것으로 판단되는 1가지만 작성하세요.
반드시 정량 근거를 포함하세요.
```

**예시:**
- ❌ 현재: "이 영상은 제주 프리미엄 레드향을 판매하는 영상입니다. 겉만 번지르르한..."
- ✅ 개선: "차별점 블록이 5개(전체의 45%)로 편중. 후기/사회적 증거 블록 추가 시 신뢰도 강화 가능."

---

### 3. 구간별 코칭 — 일반론 제거

**현재:**
Hook 구간 코칭에 "식품 카테고리에서 호기심을 빠르게 자극하고, 시청자가 어떤 내용의 영상인지 빠르게 파악하도록 하여 시청 지속률을 높이는 효과적인 방법입니다" 같은 일반론이 아직 남아 있음.

**문제:** 마케터가 이미 아는 교과서. 돈 낼 가치 없음.

**해결:** P13 프롬프트 구간별 코칭 규칙 강화.

```
구간별 코칭 규칙:

1. 절대 금지하는 문장 패턴:
   - "~하는 것이 중요합니다"
   - "~하면 효과적입니다"  
   - "~하는 방법입니다"
   - "~를 높이는 데 효과적입니다"
   이런 문장은 마케터가 이미 알고 있는 일반론입니다. 작성하지 마세요.

2. 모든 코칭에 반드시 포함할 것:
   - 해당 구간의 구체적 시간 (예: 0:03~0:07)
   - 해당 구간의 정량 데이터 (변화량, 소구 유형, 블록 타입)
   - 다른 구간 또는 영상 평균과의 비교
   
3. 제안은 구체적 행동으로:
   - ✅ "2.22초에 pain_point 블록 강화, 7.68초부터 differentiation 대신 proof 블록 삽입"
   - ❌ "공감 요소를 강화해 보세요"

4. 잘된 점/개선 포인트 각각 최대 3개. 양보다 질.
```

### 4. 타임라인 끝부분 정렬 수정

**현재:** 컬러바, 썸네일, 변화량 곡선, 스크립트 블록의 끝부분이 시간축에서 어긋남.

**문제:** 각 레이어의 width가 동일 기준에 맞지 않음.

**해결:**
- 모든 레이어(컬러바, 썸네일, 곡선, 스크립트, 소구 태그)의 width를 동일 기준으로: 영상 전체 길이 = 컨테이너 100%
- 각 요소의 left/right 위치를 `(time / total_duration) * 100%`로 통일
- 마지막 구간이 영상 끝(duration)까지 정확히 채워지도록
- 컬러바 끝, 썸네일 마지막, 곡선 x축 끝, 스크립트 마지막 블록 끝이 모두 동일 x 좌표

**확인 방법:** 영상 길이가 34초면, 0:34 지점에서 모든 레이어가 정확히 끝나야 함.

### 5. 타임라인 아래 씬별 상세 카드 추가

**현재:** 타임라인은 "위에서 전체를 훑어보는 뷰"만 존재. 특정 구간의 상세를 볼 곳이 없음. 툴팁으로 스크립트 한 줄만 표시.

**문제:** 마케터가 "이 구간 구체적으로 뭔데?"를 알고 싶으면 갈 곳이 없음.

**해결:** 타임라인 아래에 씬별 상세 카드 리스트 추가.

**레이아웃:**
```
[타임라인 — 전체 흐름 한눈에 보기]
컬러바 + 썸네일 + 변화량 곡선 + 스크립트 + 소구 태그
────────────────────────────────────────────────

[씬 상세 — 스크롤하며 상세 확인]

┌───────────────────────────────────────────────┐
│ 🔴 첫 장면  0:00~0:03                         │
│                                                │
│ ┌──────┐  스크립트:                             │
│ │ 썸네일 │  "저희 과일박스입니다"                 │
│ │      │                                       │
│ └──────┘  소구: 구성 — 정성껏 디자인된 프리미엄   │
│           선물용 패키지로, 받는 사람의 기쁨을      │
│           높여주는 특별한 경험                    │
│           전략: 경험 전환                        │
│                                                │
│           변화량: 40 (영상 평균 52 대비 낮음)     │
│           스타일: product_shot · warm             │
│                                    [▶ 재생]     │
├───────────────────────────────────────────────┤
│ 🔴 문제/공감  0:03~0:07                        │
│                                                │
│ ┌──────┐  스크립트:                             │
│ │ 썸네일 │  "겉만 번지르르한 과일이 아닐까 하는   │
│ │      │   시청자의 잠재적 의구심을 먼저          │
│ └──────┘   제기하며..."                         │
│                                                │
│           소구: 신뢰 — 두 번 이상 검수해서 당도   │
│           낮은 건 절대 보내지 않습니다             │
│           전략: 손실 회피                        │
│                                                │
│           변화량: 58 (영상 평균 대비 높음) ★      │
│           스타일: explanation · text_overlay      │
│                                    [▶ 재생]     │
├───────────────────────────────────────────────┤
│ ...                                            │
└───────────────────────────────────────────────┘
```

**카드별 정보:**

| 항목 | 내용 | 데이터 소스 |
|------|------|-----------|
| 역할 태그 + 색상 | 첫 장면 / 문제공감 / 차별점 / 증거근거 / 장점소개 / 후기반응 / 행동유도 | analysis_scenes.role |
| 시간 구간 | 0:00~0:03 | analysis_scenes.time_start/end |
| 대표 썸네일 | 씬 시작 프레임 (1~2장) | P3 frames[] |
| 스크립트 전문 | 해당 구간의 발화 텍스트 전체 | analysis_utterances (time overlap) |
| 소구 태그 | 유형 + 소구 텍스트 + 전략 | analysis_claims (time overlap) |
| 변화량 | 해당 구간 평균 + 영상 평균 대비 (높음/낮음/★ 표시) | analysis_scenes.dynamics_avg vs 전체 평균 |
| 비주얼 스타일 | shot type + color tone | analysis_scenes.style, color_tone |
| 재생 버튼 | 클릭 시 해당 구간부터 영상 재생 | time_start 기반 |

**타임라인 ↔ 씬 카드 연동:**
- 타임라인 위에서 특정 구간 클릭 → 아래 해당 씬 카드로 스크롤 + 하이라이트
- 씬 카드에서 시간 클릭 → 위 타임라인에서 해당 구간 하이라이트 + 영상 재생

**데이터 조합:**
씬별 상세 카드의 데이터는 온톨로지 시간축 겹침 쿼리로 조합:
```sql
-- 특정 씬(0:03~0:07)에 해당하는 모든 정보
SELECT * FROM get_entities_at(result_id, 3.0, 7.0);
-- → utterances + claims + blocks + cuts 전부 반환
```
온톨로지 테이블이 아직 없으면 recipe_json에서 시간 기반으로 직접 파싱.

### 6. 요약 탭 구성 정리

**현재 문제:** 영상 요약 텍스트와 핵심 코칭이 내용 중복.

**변경 후 요약 탭 구성:**

```
영상 요약
──────────
감귤  식품

[타겟] 고품질 신선 감귤에 관심이 있는 소비자 (한국어로)

이 영상은 제주 프리미엄 레드향을 판매하는 영상입니다...
(기존 요약 텍스트 유지)

길이 34초  플랫폼 upload  주스타일 프로모션  컷수 22컷  제품노출률 95%

──────────
💡 핵심 코칭
차별점 블록 5개(45%)로 편중. 후기/사회적 증거 추가 시 설득력 강화 가능.
→ 코칭 탭에서 상세 확인
──────────
```

- 영상 요약: 이 영상이 뭔지 (설명)
- 핵심 코칭: 이 영상에서 가장 고쳐야 할 한 가지 (액션)
- 두 개가 명확히 다른 역할
