## 변경 1: 히어로 헤드라인 → 로테이션 애니메이션

### 현재
```
경쟁사 숏폼,
왜 잘 되는지 알려드립니다
```
고정 텍스트. 경쟁사만 강조되어 와카랩 가치의 1/3만 전달.

### 변경
3개 헤드라인이 3초 간격으로 로테이션. 서브카피 + URL 입력창은 고정.

**헤드라인 1:**
```
이 영상은 왜 잘 되고,
내 영상은 왜 안 될까?
```

**헤드라인 2:**
```
숏폼, 감이 아니라
구조입니다
```

**헤드라인 3:**
```
분석하고, 비교하고,
더 잘 만드세요
```

**우측 이미지도 헤드라인과 동기화되어 전환:**
- 헤드라인 1 → 분석 리포트 스크린샷 (훅 강도: 강력 뱃지)
- 헤드라인 2 → 통합 타임라인 스크린샷 (시각 변화량 51 뱃지)
- 헤드라인 3 → 비교 화면 또는 제작가이드 스크린샷

**애니메이션:**
- 트랜지션: fade + slide-up (헤드라인 텍스트 + 우측 이미지 동시)
- 속도: 3초 표시 → 0.5초 트랜지션 → 3초 표시 → ...
- 하단에 인디케이터 점 3개 (●○○ → ○●○ → ○○● 순환)
- 인디케이터 클릭 시 해당 헤드라인으로 이동
- 무한 반복

**고정 요소 (애니메이션 영향 없음):**
- 상단 뱃지: COMMERCE VIDEO INTELLIGENCE
- 서브카피: "영상의 구조를 컷 단위로 해부하고, 내 영상과 비교해서, 개선 방향까지 코칭해드립니다"
- CTA 버튼 (아래 변경 2 참조)

### 온보딩 데이터 저장

**user_profiles 테이블 (신규 또는 기존 확장)**

| 컬럼 | 타입 | 설명 |
|------|------|------|
| user_id | FK → auth.users | PK |
| name | text | 가입 시 입력 |
| job_role | text | AE / 콘텐츠디렉터 / 에디터 / 대표 / 프리랜서 / 기타 |
| company_type | text | 에이전시 / 브랜드 / MCN / 프리랜서 / 기타 |
| categories | text[] | ["식품", "뷰티"] 복수 선택 |
| acquisition_channel | text | 인스타광고 / 쓰레드 / 지인추천 / 검색 / 커뮤니티 / 기타 |
| onboarding_completed | boolean | 설문 완료 여부 |
| created_at | timestamp | |

### 온보딩 완료 후
→ 메인 화면 (분석 메뉴)으로 이동
→ 첫 화면에 "분석할 영상 URL을 넣어보세요" 안내 (온보딩 가이드)

### 어드민 활용

```sql
-- 가입자 직무 분포 (타겟 검증)
SELECT job_role, COUNT(*), 
       ROUND(100.0 * COUNT(*) / SUM(COUNT(*)) OVER(), 1) as pct
FROM user_profiles
WHERE job_role IS NOT NULL
GROUP BY job_role ORDER BY count DESC;

-- 회사 유형 분포
SELECT company_type, COUNT(*)
FROM user_profiles
WHERE company_type IS NOT NULL
GROUP BY company_type ORDER BY count DESC;

-- 유입 경로 분포 (마케팅 효율)
SELECT acquisition_channel, COUNT(*)
FROM user_profiles
WHERE acquisition_channel IS NOT NULL
GROUP BY acquisition_channel ORDER BY count DESC;

-- 카테고리 관심 분포
SELECT unnest(categories) as category, COUNT(*)
FROM user_profiles
WHERE categories IS NOT NULL
GROUP BY category ORDER BY count DESC;

-- 온보딩 완료율
SELECT 
  COUNT(*) as total_signups,
  COUNT(*) FILTER (WHERE onboarding_completed = true) as completed,
  ROUND(100.0 * COUNT(*) FILTER (WHERE onboarding_completed = true) / COUNT(*), 1) as completion_rate
FROM user_profiles;
```

이 데이터가 시리즈 A에서:
- "가입자의 65%가 에이전시 AE" → 타겟 검증
- "유입의 40%가 쓰레드" → 마케팅 채널 효율
- "식품 카테고리 관심 50%" → 버티컬 포커스 근거

### 어드민 연동

어드민에 "문의 관리" 메뉴 추가:

```sql
-- 관심 서비스별 문의 분포
SELECT interest_type, COUNT(*)
FROM contact_inquiries
GROUP BY interest_type ORDER BY count DESC;

-- 최근 문의 목록
SELECT created_at, interest_type, company_name, contact_name, email, 
       LEFT(message, 100) as preview, status
FROM contact_inquiries
ORDER BY created_at DESC
LIMIT 20;

-- 전환 현황
SELECT status, COUNT(*)
FROM contact_inquiries
GROUP BY status;
```

이 데이터에서:
- "API 문의가 Enterprise의 3배" → API 사업 우선
- "마케팅 에이전시에서 Enterprise 문의 집중" → 세일즈 타겟 확인
- Atomos 같은 곳이 API 폼으로 들어올 수도 있음
