# WakaLab 비교 분석 스펙

> 2026-03-14
> Claude Code 실행용.

---

## 1. 개요

**목적:** 분석된 영상 2~3개를 비교하여 구조적 차이점/공통점 + 소구점별 표현 전략 비교 + AI 코칭 제공.

**핵심 가치:** 단순 수치 비교가 아니라 "이 소구점을 어떻게 풀어냈는지"를 비교해주는 것. 마케터가 진짜 원하는 건 "베타카로틴을 어떤 화법 + 어떤 비주얼로 표현했는지"를 영상 간 비교하는 것.

**제한:** 베타에서는 최대 3개 영상까지 비교 가능.
- Free: 월 2회
- Pro: 월 20회
- 추후 Pro에서 5개, Enterprise에서 10개로 확장 가능

---

## 2. 진입 경로 (3가지)

**경로 1: 비교 메뉴에서 직접 시작**
- 비교 화면 진입 → 영상 추가 → 비교 실행

**경로 2: 라이브러리에서**
- 라이브러리에서 영상 2~3개 체크 → "비교하기" 버튼 → 비교 화면으로 이동 (선택된 영상이 이미 추가된 상태)

**경로 3: 분석 리포트에서**
- 분석 리포트 상단에 "이 영상과 비교하기" 버튼 → 비교 화면으로 이동 (해당 영상 1개가 이미 추가된 상태) → 나머지 영상 추가

### 영상 카드 상태

| 상태 | 표시 | 동작 |
|------|------|------|
| ✅ 분석 완료 | 썸네일 + 제목 + ✅ | 비교 가능 |
| 🔄 분석 중 | 썸네일 + 제목 + 프로그레스바 | 대기 (자동 완료 감지) |
| ⚠️ 분석 필요 | 썸네일 + 제목 + "분석 필요" | 자동 분석 시작 |
| ❌ 분석 실패 | 썸네일 + 제목 + "재시도" 버튼 | 수동 재시도 |

### 분석 필요 시 처리

```
영상 추가 (URL 또는 라이브러리)
  ↓
분석 데이터 존재 확인 (results 테이블 조회)
  ├── 있음 → ✅ 분석 완료 표시
  └── 없음 → 자동 분석 시작
       ↓
     "N건의 영상 분석이 필요합니다 (분석 잔여 M/5회)"
     [N건 분석 시작]
       ↓
     동시 분석 진행 (각각 독립 job)
     전부 ✅ 완료 시 → [비교 리포트 생성] 버튼 활성화
```

**동시 분석 시 고려사항:**
- 최대 3개 영상이 동시에 분석될 수 있음
- Cloud Run maxScale 10 필요 (T13 참조)
- 분석 회수 별도 차감 (비교 회수와 독립)
  - 예: 미분석 2건 + 비교 1회 = 분석 2회 + 비교 1회 차감
- 사용자에게 분석 회수 소모 사전 안내

## 4. 시나리오 자동 결정

"개선하고 싶은 영상이 있나요?" 선택에 따라 자동 결정.

| 선택 | 시나리오 | 코칭 방향 |
|------|---------|----------|
| 특정 영상 1개 선택 | A: 내 영상 개선 | "이 영상을 나머지처럼 만들려면 이걸 바꿔라" |
| 특정 영상 여러 개 선택 | C: Variation 비교 | "이 중 구조적으로 가장 강한 건 이것" |
| "없음" 선택 | B: 패턴 추출 | "이 영상들의 공통 패턴은 이것, 이걸 적용하려면" |

**UI 로직:**
- 영상이 2개일 때: 라디오 버튼 (1개 선택 또는 없음)
- 영상이 3개일 때: 체크박스 (0~2개 선택 가능, 3개 전부 선택은 비활성)
  - 0개 선택 → 시나리오 B
  - 1개 선택 → 시나리오 A
  - 2개 선택 → 시나리오 C (2개가 내 영상 Variation, 나머지 1개가 레퍼런스? → 2개끼리 비교 + 1개는 기준)

**심플하게 가려면:**
- 영상이 몇 개든 "개선하고 싶은 영상"은 **최대 1개만 선택** 가능
- 1개 선택 → 시나리오 A (가장 흔한 케이스)
- 선택 없음 → 시나리오 B
- 시나리오 C는 명시적으로 안 나누고, "없음" 선택 시 패턴 추출로 처리 (내 영상끼리의 비교도 패턴 추출과 같은 포맷으로 충분)

**추천: 심플 버전으로 베타 시작. 시나리오 A + B만.**

## 5. 비교 리포트 — 출력 구조

### 공통 구조 (시나리오 무관)

```
비교 리포트
├── 파트 1: 전체 구조 비교 (나란히 테이블)
├── 파트 2: 소구점별 표현 전략 비교
├── 파트 3: AI 코칭 (시나리오에 따라 방향 다름)
└── 하단: [제작가이드로 이동] 버튼
```

### 파트 1: 전체 구조 비교

2~3개 영상의 content_dna 핵심 수치를 나란히 테이블.

```
                    영상 A          영상 B          영상 C
──────────────────────────────────────────────────────────
길이                32초            28초            45초
플랫폼              tiktok          tiktok          reels
컷 수               20              15              25
훅 타입             question        benefit         shock
훅 강도             strong          moderate        strong ★
첫 3초 변화량       78 ★            42              65
블록 순서           hook→pain→      hook→benefit    hook→pain→
                   auth→benefit    →cta            auth→proof→
                   →social→cta                     benefit→cta
소구 분포
  경험              58%             30%             45%
  가치              20%             50% ★           25%
  신뢰              10%             10%             20% ★
  기능              12%             10%             10%
컷 평균 길이        1.6초           1.9초           1.8초
변화량 평균         55 ★            38              48
제품 노출률         95%             72%             88%
──────────────────────────────────────────────────────────
```

**디자인 노트:**
- 각 행에서 가장 높은/좋은 값에 ★ 표시 또는 색상 강조
- 차이가 큰 항목(표준편차 큰 행) 자동 하이라이트
- 블록 순서는 시각적 플로우 바(컬러바 축소판)로도 표시

### 파트 2: 소구점별 표현 전략 비교

**이 파트가 비교 분석의 킬러 파트.**

같은 유형(또는 비슷한 주제)의 소구점을 매칭하여, 각 영상이 어떻게 표현했는지 비교.

```
┌──────────────────────────────────────────────────────┐
│  소구점 비교: "건강 효능" 관련                         │
├──────────────────────────────────────────────────────┤
│                                                       │
│  영상 A                                               │
│  ├ 소구: "한 모금이면 비타민 충전"                      │
│  ├ 유형: experience (경험)                             │
│  ├ 화법: 감성적 은유 (alpha: metaphor)                 │
│  ├ 비주얼: 클로즈업, warm 톤, sensory 스타일            │
│  ├ 구간: 0:08~0:12                                    │
│  └ [해당 구간 재생 ▶]                                  │
│                                                       │
│  영상 B                                               │
│  ├ 소구: "베타카로틴 1일 권장량의 120%"                 │
│  ├ 유형: spec (스펙)                                   │
│  ├ 화법: 수치 제시 (alpha: evidence)                   │
│  ├ 비주얼: 텍스트 오버레이, 인포그래픽                   │
│  ├ 구간: 0:05~0:08                                    │
│  └ [해당 구간 재생 ▶]                                  │
│                                                       │
│  영상 C                                               │
│  ├ 소구: "전문가가 인정한 건강 음료"                    │
│  ├ 유형: trust (신뢰)                                  │
│  ├ 화법: 권위 인용 (alpha: authority)                   │
│  ├ 비주얼: 전문가 인터뷰 컷, neutral 톤                 │
│  ├ 구간: 0:10~0:15                                    │
│  └ [해당 구간 재생 ▶]                                  │
│                                                       │
│  💡 같은 "건강 효능"을 A는 감각적 은유로, B는 수치로,   │
│     C는 전문가 권위로 풀어냄.                           │
└──────────────────────────────────────────────────────┘
```

**소구점 매칭 로직:**
1. 각 영상의 analysis_claims에서 소구점 추출
2. Gemini에 N개 영상의 소구점 목록을 넘기고 "비슷한 주제끼리 매칭하라" 지시
3. 매칭된 소구점 그룹별로 위 카드 생성
4. 매칭 안 되는 소구점(한 영상에만 있는 것)도 별도 섹션으로 표시

**데이터 소스:**
- 소구점: analysis_claims (claim, claim_type, strategy)
- 화법: analysis_blocks (alpha_emotion, alpha_structure, alpha_connection)
- 비주얼: analysis_scenes (style, visual_forms, color_tone)
- 시간축 겹침으로 소구→블록→씬 연결 (get_entities_at 함수 활용)

**매칭 프롬프트 예시:**
```
아래는 N개 영상의 소구점 목록입니다.
비슷한 주제/의도를 가진 소구점끼리 그룹으로 묶어주세요.
매칭이 안 되는 소구점은 "고유 소구"로 분류하세요.

영상 A claims: [...]
영상 B claims: [...]
영상 C claims: [...]

출력 형식 (JSON):
{
  "matched_groups": [
    {
      "theme": "건강 효능",
      "claims": [
        {"video": "A", "claim_id": "...", "claim": "...", "type": "experience"},
        {"video": "B", "claim_id": "...", "claim": "...", "type": "spec"},
      ]
    }
  ],
  "unique_claims": [
    {"video": "A", "claim_id": "...", "claim": "...", "type": "..."}
  ]
}
```

### 파트 3: AI 코칭

**시나리오 A (기준 영상 있음):**
```
📋 개선 코칭: "딸기 푸딩" 기준

1. 블록 구조
   현재: hook → benefit → cta (3단계)
   제안: hook → pain_point → authority → benefit → social_proof → cta
   근거: 영상 B, C 모두 5단계 이상 구조. 설득 단계가 많을수록 신뢰 구축에 유리.
   
2. 소구 전략
   현재: 경험 소구 58%로 편중
   제안: 신뢰 소구(전문가/근거) 추가. 영상 C처럼 권위 인용 1구간 삽입.
   구간: 0:05~0:08에 authority 블록 추가 가능.

3. 시각 표현
   현재: 0:12~0:18 구간 변화량 38 (평균 55 대비 낮음)
   제안: 영상 A의 0:08~0:12처럼 클로즈업 + 컬러 전환으로 변화량 확보.
```

**시나리오 B (패턴 추출):**
```
📋 공통 패턴 분석

이 3개 영상의 공통 구조:
• 훅: 질문형 또는 충격형 (3개 중 2개)
• 경험 소구 비율: 평균 44% (감각적 표현 중심)
• 블록 구조: 5단계 이상 (hook→pain→authority→benefit→cta 기본)
• 컷 리듬: 평균 1.8초 (빠른 리듬)
• 첫 3초 변화량: 평균 62 (높은 시각 자극)

이 패턴이 특징적인 이유:
• 식품 카테고리 평균 대비 경험 소구 비율이 14%p 높음
• 블록 단계가 평균 4단계인데 이 영상들은 5~6단계

[이 패턴으로 제작가이드 만들기 →]
```

### 하단: 제작가이드 연결

- 시나리오 A: "이 비교 결과를 바탕으로 [기준 영상]의 개선 대본 만들기"
- 시나리오 B: "이 패턴을 내 제품에 적용한 대본 만들기"
- 클릭 시 → 제작가이드 페이지로 이동. 비교 데이터가 레퍼런스로 자동 전달.

## 6. 비교 분석 API / 처리 로직

### API 콜 구조

```
비교 리포트 생성 요청
  │
  ├── 파트 1 (전체 구조 비교): API 콜 0
  │   content_dna에서 N개 조회 → 프론트에서 테이블 렌더링
  │
  ├── 파트 2 (소구점 매칭): API 콜 1 (Gemini)
  │   N개의 analysis_claims → Gemini에 매칭 요청
  │   매칭 결과 + 시간축 겹침으로 블록/씬 정보 연결
  │
  └── 파트 3 (AI 코칭): API 콜 1 (Gemini)
      N개의 recipe_json 요약 + 매칭 결과 + 시나리오 → 코칭 생성
```

**총 비용:** 비교 1건당 Gemini 2콜 ≈ $0.02~0.03
(분석 $0.05 대비 저렴 — 이미 분석된 데이터를 재활용하므로)

### 소구점 매칭 + 표현 전략 조합 로직

```
1. N개 영상의 analysis_claims 조회
2. Gemini에 소구점 매칭 요청 → matched_groups 반환
3. 각 매칭 그룹의 소구점에 대해:
   a. claim의 time_start/time_end 확인
   b. get_entities_at(result_id, time_start, time_end) 호출
   c. 해당 시간의 block (화법), scene (비주얼) 정보 연결
4. 그룹별 카드 데이터 조립
```

### 코칭 프롬프트

```
당신은 숏폼 영상 구조 전문가입니다.

[시나리오 정보]
시나리오: A (기준 영상 개선) 또는 B (패턴 추출)
기준 영상: (시나리오 A일 때만)

[분석 데이터]
영상 A 요약: {category, block_sequence, appeal_distribution, hook_type, dynamics_avg, ...}
영상 B 요약: {동일}
```

## 7. 과금 연동

### 회수 차감

| 동작 | 차감 |
|------|------|
| 비교 화면에서 미분석 영상 분석 | 분석 회수 N회 차감 |
| 비교 리포트 생성 | 비교 회수 1회 차감 |

분석과 비교는 별도 차감. 비교를 통한 분석 회수 우회 방지.

### 제한 안내 UI

```
미분석 영상 2건 발견:
"2건의 영상 분석이 필요합니다"
"분석 잔여: 3/5회 → 분석 후 1/5회"
"비교 잔여: 2/2회 → 비교 후 1/2회"
[분석 + 비교 진행]
```

Free 회수 부족 시:
```
"분석 잔여 횟수가 부족합니다 (잔여 0/5회)"
[Pro 업그레이드로 월 50회 분석 →]  (→ pro_interest_click 이벤트)
```

## 10. 구현 순서

| 순서 | 태스크 | 난이도 |
|------|--------|--------|
| 1 | 비교 입력 UI (영상 추가 + 상태 표시 + 시나리오 선택) | 중간 |
| 2 | 파트 1: 전체 구조 비교 테이블 (content_dna 기반, API 콜 없음) | 낮음 |
| 3 | 파트 2: 소구점 매칭 (Gemini 1콜 + 시간축 연결) | 높음 |
| 4 | 파트 3: AI 코칭 (Gemini 1콜, 시나리오별 프롬프트) | 중간 |
| 5 | 비교에서 미분석 영상 동시 분석 처리 | 중간 |
| 6 | 과금 연동 (분석/비교 회수 분리 차감) | 낮음 |
| 7 | comparison_reports 저장 + 이력 조회 | 낮음 |
| 8 | 진입 경로 연결 (라이브러리 → 비교, 리포트 → 비교) | 낮음 |

파트 1(구조 비교 테이블)은 content_dna만 있으면 바로 가능.
파트 2(소구점 매칭)가 핵심이자 가장 복잡한 부분 — 여기에 시간 투자.
