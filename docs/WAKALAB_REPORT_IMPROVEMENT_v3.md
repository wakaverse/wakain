## 1. 리포트 → 4탭 구조 전환

### 현재
- 모든 섹션이 세로 스크롤로 나열 (접기/펼치기 적용 상태)
- 정보 과부하 — 마케터가 전부 볼 시간 없음
- 어떤 정보에 가치를 느끼는지 파악 불가

### 변경
스크롤 나열을 제거하고, 4개 탭으로 분리.

| 탭 | 이름 | 내용 | 역할 |
|----|------|------|------|
| 1 | **요약** | 한 줄 요약 + 메타 정보(길이/플랫폼/컷수/노출률) + 핵심 코칭 1줄 | 3초 안에 결과 파악. 기본 탭. |
| 2 | **타임라인** | 통합 타임라인 (설득 흐름 컬러바 + 시각 변화량 곡선 + 스크립트 + 소구 태그 + 썸네일) | 시간 순서로 영상 파악 |
| 3 | **코칭** | 잘된 점 + 개선 포인트 + 추천 구조(현재 vs 제안) + 제작가이드 버튼 | "뭘 고치라고?" |
| 4 | **구조 분석** | 콘텐츠 포지셔닝 + 훅 분석 + 제품 소구 분석 | 깊이 파고 싶은 사람용 |

### 기존 섹션 → 탭 매핑

| 기존 섹션 | 이동할 탭 |
|----------|---------|
| 영상 요약 | 탭1: 요약 |
| 핵심 코칭 (요약 카드 내) | 탭1: 요약 |
| 통합 타임라인 | 탭2: 타임라인 |
| 코칭 — 잘된 점 | 탭3: 코칭 |
| 코칭 — 이렇게 하면 더 좋아요 | 탭3: 코칭 |
| 코칭 — 추천 구조 | 탭3: 코칭 |
| 코칭 — 구간별 코칭 (Hook/Body/CTA) | 탭3: 코칭 |
| 제작가이드 버튼 | 탭3: 코칭 (하단) |
| 콘텐츠 포지셔닝 | 탭4: 구조 분석 |
| 훅 분석 | 탭4: 구조 분석 |
| 제품 소구 분석 | 탭4: 구조 분석 |

### 제거 항목
- 체크리스트 — 프론트 비노출 (P13에서 데이터는 계속 생성, 프론트에서만 제거)

### 탭 클릭 로그
- 탭 전환 시 이벤트 기록
- 형식: `{ action: "tab_click", metadata: { tab_name: "coaching" }, result_id: "xxx" }`
- 저장: user_activity_logs 테이블 (테이블 없으면 생성)
- 목적: 베타 후 "마케터들은 어떤 탭을 가장 많이 보는가" 데이터 확보

## 2. 코칭 품질 — P13 프롬프트 개선

### 현재 문제
코칭이 일반론으로 가득함. 예시:
- ❌ "숏폼에서는 초반에 제품을 빨리 노출하는 것이 중요합니다"
- ❌ "시각적 변화가 빠르면 이탈을 줄일 수 있습니다"
- ❌ "CTA에 긴급성을 더하면 좋습니다"
→ 마케터가 이미 아는 교과서. 돈 낼 가치 없음.

### 개선 원칙

**금지:**
- "~하는 것이 중요합니다" 형태의 일반론
- "~하면 좋습니다" 형태의 뻔한 제안
- 이 영상 데이터와 무관한 일반 숏폼 팁

**필수:**
- 모든 코칭에 이 영상의 정량 데이터 근거 포함
- 구간 지정 (몇 초~몇 초)
- 비교 기준 제시 (카테고리 평균 대비, 또는 영상 내 다른 구간 대비)

### 개선 예시

**잘된 점:**
- ❌ 기존: "전체 영상에서 제품 노출 비중이 95%로 매우 높았습니다."
- ✅ 개선: "제품 노출률 95%. 식품 카테고리 평균 72% 대비 23%p 높음. 0초부터 제품이 등장해 인지 지연 없음."

**개선 포인트:**
- ❌ 기존: "영상에 '이런 고민 있죠?'와 같은 시청자의 실제 고민을 시각적으로 보여주는 장면이 부족합니다. ... 비싼 디저트 값에 놀라거나, 아이 간식으로 무엇을 줄지 고민하는 엄마의 모습을 짧게라도 연출하여 공감대를 강화해 보세요."
- ✅ 개선: "0:02~0:05 문제/공감 구간: 텍스트 멘트만 존재, 시각적 공감 장면 없음. 이 구간 시각 변화량 42 (영상 평균 55 대비 낮음). 제안: 고민 상황 시각화 1~2초 삽입 시 변화량 + 공감 동시 확보 가능."

### P13 프롬프트에 추가할 지시문 (예시)

```
코칭 작성 규칙:
1. 일반적인 숏폼 제작 조언은 절대 하지 마세요. 사용자는 이미 알고 있습니다.
2. 이 영상의 분석 데이터에서만 발견되는 구체적 사실만 코칭하세요.
3. 모든 코칭 항목에 반드시 포함:
   - 해당 구간 시간 (예: 0:05~0:12)
   - 정량 수치 (예: 시각 변화량 38, 경험 소구 비율 58%)
   - 비교 기준 (예: 영상 평균 대비, 카테고리 평균 대비)
4. "~하는 것이 중요합니다", "~하면 좋습니다" 형태의 문장은 사용하지 마세요.
5. 제안은 구체적 행동으로: "공감 장면 추가" (O), "공감대를 강화해 보세요" (X)
6. 잘된 점은 3개 이내, 개선 포인트는 3개 이내. 양보다 질.
```

### 코칭 텍스트 양 목표
- 현재 대비 절반 이하
- 잘된 점: 최대 3개, 각 2~3줄
- 개선 포인트: 최대 3개, 각 2~3줄 (구간 + 수치 + 제안)
- 구간별 코칭 (Hook/Body/CTA): 각 2~3줄

## 3. 통합 타임라인 개선

### 3-1. 썸네일 뷰 추가

**현재:** 컬러바 + 변화량 곡선 + 스크립트 텍스트(잘림) + 소구 태그
**추가:** 씬별 대표 썸네일

**구현:**
- 컬러바와 변화량 곡선 사이, 또는 컬러바 바로 아래에 썸네일 행 추가
- 각 씬의 대표 프레임 (P3에서 추출된 frames[] 중 씬 시작 프레임)
- 타임라인 비율에 맞게 배치 (씬 길이에 비례한 너비)
- 텍스트 뷰 / 썸네일 뷰 토글 버튼, 또는 둘 다 표시

**데이터 소스:** P3 frames[] (이미 존재), recipe_json.scenes[].time_range

### 3-2. 스크립트 텍스트 잘림 개선

**현재:** 가로 공간에 블록 텍스트가 우겨넣어져서 "체...", "체..." 로 잘림
**현재 이미 있는 것:** 마우스 호버 시 툴팁으로 전체 텍스트 표시, 클릭 시 해당 지점 영상 재생

**개선 방향:**
- 텍스트 라벨에 블록 유형만 표시 (예: "가치", "신뢰", "경험", "기능")
  → 현재 "이거 모르고 디저트..." 대신 소구 태그 아이콘 + 유형명만
- 상세 스크립트는 툴팁(이미 있음) + 클릭 시 하단 패널에 표시
- 또는: 타임라인 아래에 "현재 선택된 구간" 상세 패널 (스크립트 + 소구 + 썸네일)

## 4. 제품 소구 분석 개선

### 현재 문제
가치 2개, 신뢰 1개, 경험 7개, 기능 2개가 리스트로 쭉 나열.
정보는 많은데 "그래서 뭐?" 가 없음.

### 변경

**상단: 소구 비율 차트 + 한 줄 해석**
- 원형 차트 또는 가로 막대 차트: 유형별 비율 시각화
- 한 줄 해석 예시: "경험 소구 58% (7건) — 감각적 묘사 중심. 가치/신뢰 소구는 상대적으로 부족."
- (추후 벤치마크 연동 시: "식품 카테고리 평균 경험 소구 40% 대비 18%p 높음")

**하단: 개별 소구 상세 (펼치기)**
- 기본 접힘 상태
- 소구 유형별 그룹핑 (경험: 7건, 가치: 2건, 신뢰: 1건, 기능: 2건)
- 펼치면 현재와 같은 개별 소구 리스트

**데이터 소스:** product.claims (이미 존재), claim_type별 집계

## 6. 요약 탭 정보 보강 (2차 개선)

### 현재 문제
- 요약에 핵심 수치가 부족하여 비교/제작가이드로 넘어가기 전 판단이 어려움
- 콘텐츠 포지셔닝이 구조 분석 탭에만 있음 — 요약에서 안 보임
- 제품을 왜 사야 하는지 요약에서 안 보임 — 구조 분석 탭까지 가야 함
- 인물 등장 정보가 리포트 어디에도 안 나옴 (P2에서 추출하지만 미표시)
- "experience 중심 demo형" 같은 내부 구조 표현이 그대로 노출

### 변경 — 요약 탭 전체 구조

```
영상 요약

골지 브라탑  BENETTON  패션
편안하고 스타일리시한 이너웨어를 찾는 여성

[핵심 수치 카드 — 가로 5개]
┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐
│훅 강도  │ │변화량 ⓘ│ │소구 Top │ │블록 수  │ │노출률   │
│ 보통   │ │  17    │ │경험 42% │ │  12개  │ │  95%  │
└────────┘ └────────┘ └────────┘ └────────┘ └────────┘

인물: 있음 (호스트, 얼굴 노출)
길이 47.5초  플랫폼 upload  주스타일 데모  컷수 3컷

── 콘텐츠 포지셔닝 ──
이너웨어의 편안함과 겉옷의 스타일리시함을 결합하여,
하나의 제품으로 속옷 착용의 불편함을 해소하고 스타일링까지 가능함을 보여줌.

── 제품을 사야 하는 이유 ──
• 쫀쫀하지만 답답하지 않은 착용감 (경험)
• 패드 고정으로 움직여도 안정적 (기능)
• 속옷 없이 브라탑 하나로 코디 완성 (기능)
• 베네통 코리아 정품 (신뢰)
→ 핵심 설득: "편안함 + 스타일 = 속옷 없이도 되는 브라탑"

── 구조 흐름 ──
훅 → 장점 → 시연 → 장점 → 문제/공감 → 시연 → 장점 ... → 행동유도

💡 핵심 코칭
행동 유도(CTA) 블록이 누락되어 구매 전환 기회를 놓치고 있습니다...
→ 코칭 탭에서 상세 확인
```

### 추가 요소 상세

**핵심 수치 카드:**
- 비교 목록에 넣을지 말지를 요약만 보고 판단 가능하도록
- 변화량 옆 ⓘ 아이콘 → 클릭 시 툴팁 (아래 T8 참조)
- 카드 색상: 기준 대비 좋으면 초록, 나쁘면 빨강, 보통이면 회색

**인물 등장 정보:**
- P2 human_presence 데이터를 요약에 표시
- "있음 (호스트, 얼굴 노출)" / "있음 (모델, 얼굴 미노출)" / "없음 (제품만)"
- 비교/content_dna 혼동 변수(has_person, person_role, face_visible)와 연결

**콘텐츠 포지셔닝:**
- 현재 구조 분석 탭에만 있음 → 요약에도 표시 (구조 분석의 포지셔닝 데이터를 요약에서 1~2줄로 압축)
- 차별 각도 + 대비 구조의 핵심만

**제품을 사야 하는 이유:**
- P7(PRODUCT) claims에서 유형별 대표 소구 1개씩 추출
- 또는 P13에서 별도로 "이 제품의 핵심 구매 이유 3~4개"를 생성
- 마지막에 "핵심 설득" 한 줄 — 이 영상의 전체 소구를 한 문장으로 압축
- 이것이 비교할 때 "이 영상의 셀링 포인트"로 바로 읽힘

**구조 흐름:**
- 블록 순서를 한 줄로 표시 (한국어로)
- "experience 중심 demo형" → "경험 소구 중심의 시연형 영상"으로 번역
- 영어 블록 타입 → 한국어 (hook→훅, benefit→장점, demo→시연, cta→행동유도 등)

**데이터 소스:**
- 핵심 수치: content_dna 또는 recipe_json에서 직접
- 인물: P2 human_presence
- 포지셔닝: evaluation.positioning (P13)
- 소구 요약: product.claims (P7) 집계 + P13에서 "핵심 설득" 생성
- 구조 흐름: script.blocks 순서

## 7. P13 코칭 — 영상 맥락 인식 + UI 개선 (2차 개선)

### 7-1. 영상 맥락별 코칭 방향 조정

**현재 문제:**
모든 영상에 "시각 변화량을 높여라"로 코칭. 근데 GS리테일 홈쇼핑 재활용 영상처럼 시연/간증 중심의 영상에서는 변화량이 낮은 게 문제가 아닐 수 있음. 구매 의향이 있는 타겟에게는 시각적 자극보다 신뢰 구축이 더 중요.

**P13 프롬프트 추가:**
```
코칭 맥락 규칙:
- 영상의 목적과 타겟 맥락에 따라 코칭 방향을 조정하라.
- 모든 영상에 높은 시각 변화량이 필요한 것은 아니다.
- 플랫폼이 upload이고 스타일이 데모이고 주요 소구가 기능/경험 중심이면:
  → "시각 변화량을 높여라"가 아니라 "시연 구간의 설득력을 높여라"로 코칭.
- 틱톡/릴스 네이티브 영상과 홈쇼핑/쇼핑 재활용 영상은 코칭 방향이 달라야 한다.
- 타겟이 구매 의향이 높은 사람(앱 내 시청)이면 훅보다 신뢰/증거/CTA 강화 중심.
```

### 7-2. 내부 필드명 노출 제거

**현재 문제:**
코칭에 "product_exposure_pct", "script_alpha_summary", "checklist_results", "pain_point", "cta" 같은 내부 필드명이 그대로 노출.

**P13 프롬프트 추가:**
```
출력 규칙:
- 영어 필드명, 내부 변수명, 데이터 구조명을 절대 노출하지 마라.
- product_exposure_pct → "제품 노출 비중"
- script_alpha_summary → 해당 내용을 한국어로 설명
- checklist_results → "분석 결과"
- pain_point → "고민/문제 제기"
- cta → "행동 유도"
- social_proof → "사회적 증거"
- differentiation → "차별점"
- benefit → "장점/매력"
- authority → "전문가/권위"
- 모든 블록 타입, 소구 유형, 전략명을 한국어로 번역하여 출력.
```

### 7-3. 코칭 UI 개선 — 결과 강조 + 명칭 변경

**현재 문제:**
- 코칭의 핵심 결과(개선 포인트)가 시각적으로 안 띔 — 잘된 점과 개선 포인트의 시각적 차이 부족
- "추천 구조"라는 명칭이 애매 — 이것이 "개선된 구조"라는 게 직관적으로 안 읽힘

**변경:**

**a) "추천 구조" → "구조 개선안"으로 명칭 변경**
```
현재: 📎 추천 구조
변경: 📎 구조 개선안
```
"추천"은 "참고해봐"느낌. "개선안"은 "이렇게 바꿔라"느낌. 후자가 코칭에 맞음.

**b) 현재 구조 vs 개선안 시각적 차이 강화**
```
현재:
  현재
  첫 장면 → 할인/혜택 → 후기/반응 → 차별점 → ...
  
  제안
  hook → benefit → social_proof → ...

변경:
  ❌ 현재 구조
  훅 → 할인 → 후기 → 차별점 → 문제 → 차별점 → 행동유도
  (배경: 연한 빨강/회색)
  
  ✅ 개선안
  훅 → 고민제기 → 해결책 → 증거 → 희소성 → 행동유도
  (배경: 연한 초록)
  
  왜 이렇게 바꿔야 하나:
  현재 영상은 ... (1~2줄 근거)
```

**c) 개선 포인트 시각적 강조**
```
현재: 💡 이렇게 하면 더 좋아요 (일반 텍스트)
변경: 
┌─────────────────────────────────────────┐
│ 🔧 핵심 개선 포인트                       │ ← 배경색 있는 카드
│                                          │
│ 1. 고민/공감 장면 부족                     │
│    11.76초 차별점 구간에 공감 장면 추가     │
│                                          │
│ 2. 증거/근거 불명확                        │
│    재료 간단함, 조리 시간 등 인포그래픽 추가 │
│                                          │
│ 3. CTA 누락                              │
│    마지막 3초에 명확한 행동 유도 추가        │
└─────────────────────────────────────────┘
```

개선 포인트는 번호 + 제목 + 구체적 행동 1줄. 배경색 있는 카드로 시각적 분리.
잘된 점은 현재처럼 ✅ 리스트. 개선 포인트만 카드로 강조하면 차이가 생김.

## 8. 시각 변화량 설명 추가

### 현재 문제
"시각 변화량 17"이라는 숫자가 마케터에게 생소. 높은 건지 낮은 건지, 뭘 의미하는 건지 모름.

### 변경

**a) 요약 탭 — 변화량 카드 옆 ⓘ 툴팁**

```
시각 변화량: 17 ⓘ

ⓘ 클릭 시 툴팁:
┌────────────────────────────────────────────┐
│ 시각 변화량 (Visual Dynamics)               │
│ WakaLab 독자 지표                          │
│                                            │
│ 화면이 얼마나 역동적으로 변하는지를           │
│ 측정한 수치입니다.                           │
│ 컷 전환, 움직임, 색감 변화가 클수록          │
│ 높아집니다.                                 │
│                                            │
│ 50 이상  역동적 (시선 집중 높음)             │
│ 30~50   보통                               │
│ 30 이하  정적 (이탈 위험 가능)               │
│                                            │
│ ※ 시연/데모 영상은 낮아도 정상일 수 있음      │
└────────────────────────────────────────────┘
```

첫 방문 시 또는 처음 분석 결과를 볼 때 한 번 자동으로 툴팁이 뜨게 하면 더 좋음 (온보딩 가이드 성격).

**b) 타임라인 씬별 상세 — 변화량 한 줄 해석**

```
현재: 변화량: 66 (평균 46 대비 높음)
변경: 변화량: 66 (평균 46 대비 높음)
     → 이 구간은 화면 전환이 빠르고 시각 자극이 강합니다

현재: 변화량: 10 (평균 17 대비 낮음)
변경: 변화량: 10 (평균 17 대비 낮음)
     → 이 구간은 정적인 화면이 이어지고 있습니다
```

숫자만이 아니라 "그래서 뭐?"까지 한 줄로.

## 9. 타임라인 씬별 상세 개선

### 현재 문제
- 소구 태그가 많이 나열되는데 "이 씬이 잘한 건지 못한 건지" 판단 어려움
- 소구 리스트가 길어서 핵심이 묻힘

### 변경

**씬 카드 기본 표시:**
```
시연/사용법  0:08~0:47                              ▷ 재생
┌──────┐  "제가 입은 모습 바로 보여드릴게요..."
│ 썸네일 │  
│      │  💡 제품 착용 시연을 통해 편안함과 핏을 직접 보여주는 구간.
└──────┘     소구 10개로 가장 밀도 높은 구간이나, 변화량 17로 시각적 단조로움.
          
          변화량: 17 (평균 대비 수준) · medium · neutral
          소구: 경험 5건 / 기능 4건 / 신뢰 1건  [펼치기 ▼]
```

**변경 포인트:**
- 씬마다 **한 줄 평가**(💡) 추가 — 이 씬의 역할 + 핵심 특징 + 문제점을 한 문장으로
- 소구 개수만 유형별로 표시, 상세 리스트는 [펼치기]로
- 한 줄 평가는 P13에서 생성 또는 P8+P10 데이터에서 조합

**한 줄 평가 생성 규칙:**
```
P13 프롬프트 추가:
각 씬에 대해 1줄 평가를 생성하라:
- 이 씬의 역할 (시연/설득/증거/공감 등)
- 핵심 특징 1개
- 문제점 또는 강점 1개
- 총 30~50자 이내
```

## 10. 제품 소구 분석 — 접기 기본 상태 + UI

### 현재 문제
- 소구 유형별 상세 리스트가 기본 접힌 상태 → 눌러야 보임
- 유형별 대표 소구가 뭔지 접지 않고는 모름

### 변경

**차트 아래에 유형별 대표 소구 1줄씩은 기본 표시:**

```
제품 소구 분석
제품 노출률 95%  제품 첫 등장 0.5초

경험 ████████████████████ 42%  5건
신뢰 ██████████         22%  4건  
가치 ███████            17%  3건
구성 █████              11%  2건
기능 ███                 6%  1건

경험 소구 42% (5건) — 경험 중심 구성. 신뢰 소구는 상대적으로 부족.

대표 소구:
• 경험 — "쫀쫀하고 답답하지 않음" (착용감)
• 기능 — "패드 고정, 움직이지 않음" (안정성)
• 신뢰 — "베네통 코리아 정품" (브랜드)
• 가치 — "비밀스러운 가격" (프로모션)

[전체 소구 상세 보기 ▼]  ← 클릭 시 현재와 같은 유형별 전체 리스트 펼침
```

차트 + 한 줄 해석 + 대표 소구 4줄 → 이것만 봐도 "이 영상이 뭘로 설득하는지" 파악 가능.
전체 상세(경험 5건, 기능 4건...)는 펼치기 안에.
