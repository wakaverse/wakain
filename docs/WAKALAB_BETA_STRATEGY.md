## 1. 한 줄 정의

**"숏폼 영상이 왜 잘 되는지 구조적으로 분석하고, 어떻게 고쳐야 하는지 코칭하고, 잘 되는 구조로 새 영상을 만들 수 있게 해주는 콘텐츠 인텔리전스 플랫폼"**

## 2. 포지셔닝

| 맥락 | 포지셔닝 |
|------|---------|
| 대외 (마케터) | 커머스/마케팅 영상 분석 전문 |

## 3. 타겟

**1차 타겟: 마케팅 에이전시 AE (Account Executive)**

- 클라이언트한테 "왜 이 영상이 안 되는지" 설명해야 하는 사람
- "다음 영상은 이렇게 만들겠습니다" 제안서를 써야 하는 사람
- 와카랩 리포트가 그대로 클라이언트 보고용이 됨
- 이 사람이 써보고 좋으면 팀에 전파 → 팀장이 결제

**리포트 목적별 대응:**
- 외부 공유용 (클라이언트 보고): 요약 + 코칭 핵심 + 추천 구조만 깔끔하게
- 내부 검토용: 전체 데이터 날것 그대로
- 리포트 내보내기(PDF/링크) 시 템플릿 선택으로 구현

## 4. 핵심 가치 — 3단계 루프

```
① 분석: "이 영상이 왜 잘 되는지" 구조적으로 해부
     ↓
② 코칭: "내 영상은 어떻게 고쳐야 하는지" 데이터 기반 제안
     ↓
③ 제작: "잘 되는 구조로 내 제품 영상 만들기" 가이드 + 대본
     ↑__________________________________________________↩ (반복)
```

## 5. 온보딩 — 첫 방문 시나리오

```
첫 화면: URL 입력창 하나
"분석할 영상 URL을 넣어보세요"
     ↓
분석 중 (2~3분)
     ↓
분석 완료 → 리포트 4탭 표시
     ↓
자동 안내:
├── "이 영상을 라이브러리에 저장했어요"
├── "경쟁사 영상과 비교해보세요" → 비교 유도
├── "이 구조로 제작가이드 만들어보세요" → 제작가이드 유도
└── "이 채널의 새 영상을 자동으로 트래킹할 수 있어요" → 레이더 유도
```

메뉴 5개를 먼저 보여주지 않음. 첫 경험은 URL 입력 한 방.
메뉴는 있되, 자연스럽게 발견하게 유도.

## 6. 메뉴 구조 — 5개 (동시 오픈)

### 6-1. 분석
**역할:** 영상 → 13단계 분석 → 리포트 → 라이브러리 자동 저장
**입력:** 파일 업로드 / URL
**출력:** 4탭 리포트

| 탭 | 내용 | 역할 |
|----|------|------|
| 요약 (기본) | 한 줄 요약 + 메타 + 핵심 코칭 1줄 | 3초 파악 |
| 타임라인 | 컬러바 + 변화량 곡선 + 스크립트 + 썸네일 | 시간순 파악 |
| 코칭 | 잘된 점 + 개선 + 추천 구조 + 제작가이드 버튼 | "뭘 고치라고?" |
| 구조 분석 | 포지셔닝 + 훅 + 소구 분석 | 깊이 파기 |

분석 완료 시 라이브러리 자동 저장.

상세: WAKALAB_REPORT_IMPROVEMENT.md 참조.

### 6-2. 비교
**역할:** 분석된 영상 N개를 비교해서 인사이트 도출
**입력:** 라이브러리에서 N개 선택
**출력 (A+B+C, 베타):**
- A) 구조 차이 — 블록 순서 나란히 비교
- B) 수치 비교 — 훅 강도, 리듬 속도, 시각 변화량, 소구 분포
- C) AI 코칭 — "이 영상이 저 영상보다 나은 이유" + 개선 제안

**추후 추가 (D, 데이터 축적 후):**
- D) 공통 패턴 — "이 3개 영상의 공통 패턴은 질문형 훅 + 빠른 리듬"

**핵심:** 이종권 CBO 피드백 — "잘되는 영상 vs 내 영상 비교 → 코칭이 핵심"

### 6-3. 레이더
**역할:** 경쟁 채널 모니터링 + 새 영상 감지 → 자동 분석 → 라이브러리에 저장
**입력:** 채널 URL 등록 (틱톡/유튜브/인스타)
**동작:**
- 등록된 채널의 최근 영상 N개 자동 분석
- 초기: 수동 새로고침 OK, 스케줄러는 후순위
- 새 영상 감지 시 인앱 알림
**핵심:** 매일 들어오는 이유. 리텐션 엔진.

### 6-4. 라이브러리
**역할:** 분석된 영상 저장/관리/검색. 비교의 입구, 레이더의 출구.
**필터:** 카테고리별, 플랫폼별, 브랜드별, 훅 강도별
- 필터가 content_dna 컬럼과 매핑
**자동 저장:** 분석 완료 시, 레이더 감지 시

### 6-5. 제작가이드
**역할:** 분석 기반 영상 제작 방향 + 대본 초안 생성
**모드 A (핵심):** 레퍼런스 참고형
- 분석된 영상의 구조를 참고 → 내 제품에 맞게 변환 → 대본 초안
- 코칭 탭 하단 "제작가이드" 버튼으로 진입
**모드 B:** 직접 입력형
- 제품정보 + 목적 + 영상 소스 → 가이드 생성
- 모드 A 없이도 독립 사용 가능

**리포트 출력:**
- 구조 개선 (Hook/Body/CTA)
- 레시피 리팩토링 (현재 vs 제안)
- 핵심 개선 포인트
- 대본 생성 버튼

## 7. 과금 모델 — Free / Pro

| 기능 | Free | Pro |
|------|------|-----|
| 분석 | 월 5회 | 월 50회 |
| 비교 | 월 2회 | 월 20회 |
| 라이브러리 | 20개 저장 | 200개 저장 |
| 레이더 | 채널 1개 | 채널 5개 |
| 제작가이드 | 월 3회 | 월 30회 |
| 대본 생성 | 월 1회 | 월 10회 |
| 리포트 내보내기 | 워터마크 있음 | 워터마크 없음 + 템플릿 선택 (외부용/내부용) |

**설계 원칙:**
- Free에서 가치를 충분히 체험 (분석 5회, 비교 2회면 "이게 뭔지" 파악 가능)
- "계속 쓰려면" Pro 필요 (자연스러운 업그레이드 동기)
- Pro도 무제한 아님 (Enterprise 티어 여지)
- 워터마크는 자연 홍보 + Pro 전환 동기 동시 달성

**결제 의향 테스트 방식:**
- Pro 기능 제한 도달 시 팝업: "Pro로 업그레이드하면 월 50회까지 분석할 수 있어요"
- "관심 있어요" 버튼 클릭률 추적
- 클릭한 사람에게 "얼마면 쓰시겠어요?" 설문
- 베타 기간에는 실제 결제 없음

**기능별 회수 관리:**
- JSONB로 유연하게 (하드코딩 X)
- 월 초 리셋
- 어드민에서 특정 사용자 회수 조정 가능 (테스트/VIP용)

## 8. 사용자 획득 — 월 300만원

### Phase 1: 커뮤니티 시딩 (비용 0원, 오픈 전~1주차)

**채널:**
- 쓰레드 — "숏폼 해부 시리즈" 매일 1건. 실제 잘 되는 영상을 와카랩으로 분석한 결과 캡처.
- 마케터 오픈카톡방, 페이스북 마케팅 그룹, 디지털 마케팅 슬랙
- "무료로 숏폼 분석해드립니다" 게시

**목적:**
- 어떤 메시지에 반응하는지 테스트
- "경쟁사 분석" vs "내 영상 코칭" vs "잘 되는 영상의 비밀" 중 먹히는 카피 확인

### Phase 2: 인스타 광고 (하루 5만원, 2주차~)

**예산:** 월 ~150만원
**시딩에서 검증된 메시지로 광고 크리에이티브 제작**
**크리에이티브:** 와카랩 분석 결과 스크린샷 활용
- "이 틱톡 영상 500만뷰의 비밀을 분석해봤습니다"
- "경쟁사 숏폼, 왜 잘 되는지 알려드립니다"
**타겟:** 마케팅 직무, 에이전시 관련, 숏폼/콘텐츠 관심사
**예상:** CPC 3,000원 기준 → 월 500클릭 → 가입 50~100명

### Phase 3: 콘텐츠 + 협업 (나머지 ~150만원)

- 마케팅 유튜버/인플루언서에게 "와카랩으로 본인 영상 분석해보는 콘텐츠" 의뢰
- 와카랩 분석 결과 자체를 콘텐츠로 제작 → 쓰레드/인스타 업로드

## 11. 기술 판단

- Python 아키텍처 유지 (일 10,000건까지 충분)
- 병목 시 큐/캐싱/DB 최적화로 해결 (언어 변경 불필요)
