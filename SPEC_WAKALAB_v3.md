# SPEC_WAKALAB_v3.md — 와카랩 제품 스펙

**버전:** v3 (2026-03-02)
**상태:** 컨셉 리뉴얼 — "영상 분석 도구"에서 "커머스 숏폼 해킹 플랫폼"으로 전환

---

## 1. 제품 정의

### 한 문장
> 떡상한 커머스 영상을 해킹해서, 내 제품에 맞는 소재를 만들어주는 플랫폼

### 핵심 철학
- 우리는 "이 영상이 팔릴지"를 **판정하지 않는다**
- 우리는 "이 영상이 어떻게 생겼는지"를 **해부해서 보여준다**
- 판단은 사용자가 한다. 우리는 구조를 보여주고, 다음 행동을 도와준다

### 왜 판정하지 않는가
영상의 성과에는 영상 외적 변수(인플루언서 영향력, 플랫폼 알고리즘, 제품 자체 경쟁력, 게시 타이밍, 브랜드 인지도 등)가 영상 내적 요소보다 크게 작용함. 따라서 영상 구조만으로 "팔릴지 안 팔릴지"를 판정하는 것은 불가능하며, 해서도 안 됨.

### 경쟁 포지셔닝
- **Sandcastles.ai** (74K유저, $49~$499/월): "바이럴(조회수) 크리에이터의 리서치+스크립트 도구"
- **WakaLab**: "커머스(전환) 마케터+크리에이터의 해킹+소재 도구"
- 같은 워크플로우(발견→분석→제작), 다른 도메인(바이럴 vs 커머스), 다른 깊이(영상 전체 vs 컷 단위)

## 2. 용어 체계

| 한국어 | 영어 | 의미 |
|--------|------|------|
| **떡상** | **Spike** | 성과가 뾰족하게 튄 영상 |
| **해킹** | **Hack** | 영상 구조를 컷 단위로 분해·분석 |
| **레시피** | **Recipe** | 해킹 결과를 재현 가능한 패턴으로 정리 |
| **레이더** | **Radar** | 카테고리/브랜드별 떡상 영상 모니터링 |

## 3. 타겟 사용자

### Segment 1: 쇼핑 숏폼 크리에이터

**프로필:**
- 1인 또는 소규모
- 직접 촬영·편집·업로드
- 수익 = 조회수 x 전환 (쿠팡파트너스 커미션 등)
- 매일 1~3개 영상 제작
- 핵심 고민: "어떤 영상을 만들어야 돈이 되는가?"

**사용 흐름:**
```
레이더 → 해킹 → 대본 → 소재확장
"뭐가 뜨지?" → "어떻게 생겼지?" → "대본 써줘" → "다른 버전도"
```

**핵심 가치:**
- 뭘 찍을지 알려줌 (레이더)
- 어떻게 찍을지 알려줌 (해킹 → 대본)
- 같은 제품으로 여러 소재 (소재확장)

### Segment 2: 마케터 (에이전시/브랜드)

**프로필:**
- 팀 단위 (마케터 + PD + 편집자)
- 여러 제품 동시 운영
- 성과 보고 필요 (CTR, 조회수 등)
- 영상은 직접 안 만듦, 기획·발주·검수
- 핵심 고민: "경쟁사 대비 우리 영상이 어떤지?"

**사용 흐름:**
```
레이더 → 해킹 → 비교 → 인사이트 → 소재확장
"경쟁사 뭐하지?" → "분석" → "우리랑 비교" → "트렌드" → "다음 소재"
```

**핵심 가치:**
- 경쟁사 파악 (레이더 + 워치리스트)
- 구조적 비교 (비교)
- 데이터 기반 기획 (인사이트)
- 다음 소재 방향 (소재확장)

### 4-1. 레이더 (Radar)

**목적:** "지금 뭐가 뜨고 있는가?"

#### 서브메뉴

| 서브 | 기능 |
|------|------|
| **카테고리 피드** | 카테고리별(뷰티/식품/가전/패션 등) 떡상 영상 자동 수집·정렬 |
| **워치리스트** | 특정 브랜드/채널 팔로우, 신규 영상 알림 |

#### 화면 구성

```
[카테고리 피드]  [워치리스트]

카테고리: [뷰티 v] [식품] [가전] [패션] [건강] ...
정렬: [Spike 배수] [조회수] [최신]

┌────────────┐ ┌────────────┐ ┌────────────┐
│ Spike 12.3x │ │ Spike 8.7x  │ │ Spike 6.2x  │
│ [썸네일]     │ │ [썸네일]     │ │ [썸네일]     │
│ 선크림 · 뷰티│ │ 립 · 뷰티   │ │ 세럼 · 뷰티 │
│ 조회수 45만  │ │ 조회수 23만  │ │ 조회수 18만  │
│ [해킹하기]   │ │ [해킹하기]   │ │ [해킹하기]   │
└────────────┘ └────────────┘ └────────────┘
```

#### 시나리오

**크리에이터 — "오늘 뭐 찍지?"**
1. 레이더 접속, 뷰티 카테고리 선택
2. 이번 주 떡상 영상 목록 확인 (Spike 배수 + 썸네일 + 조회수)
3. "선크림 영상이 떡상했네" → [해킹하기] 클릭
4. → 해킹 메뉴로 이동

**마케터 — "경쟁사 뭐 하고 있지?"**
1. 워치리스트에 경쟁 브랜드 3개 등록해둠
2. 레이더 접속, 워치리스트 탭
3. "올리브영이 새 영상 올렸네" → [해킹하기] 클릭
4. → 해킹 메뉴로 이동

#### 기술 요구사항
- SNS 스크래핑 Agent (인스타/틱톡/유튜브 쇼츠 공개 영상)
- 카테고리 자동 분류 (제품/브랜드 태깅)
- Spike 배수 계산 (채널 평균 대비 조회수)
- 개발 우선순위: 2차

### 4-2. 해킹 (Hack)

**목적:** "이 영상은 어떻게 생겼는가?"

#### 서브메뉴 (탭)

| 탭 | 기능 |
|---|---|
| **해킹 결과** | 훅/소구/편집/레시피 카드 (Sandcastles식 직관적 카드) |
| **컷 뷰** | 컷별 타임라인 상세 (WakaLab 고유 차별점) |

#### 입력 방식
- 영상 파일 업로드
- URL 입력 (인스타/틱톡/유튜브)
- 레이더에서 [해킹하기] 클릭

#### 해킹 결과 탭 — 카드형 (첫 화면)

```
┌─────────────────────────────────────────────┐
│                                             │
│  [영상 플레이어]                              │

│                                             │
│  "시연훅 → 기능소구x3 → 가격반전 CTA"        │
│                                             │
├─────────────────────────────────────────────┤
│                                             │
│  [해킹 결과]  [컷 뷰]                        │
│                                             │
├─────────────────────────────────────────────┤
│                                             │
│  훅                                         │
│  시연형 — 제품 사용 장면으로 바로 시작        │
│                                             │
├─────────────────────────────────────────────┤
│                                             │
│  소구 포인트                                 │
│  (1) 기능 — 자외선 차단율 SPF50              │
│  (2) 비교 — "타사는 만원인데"                 │
│  (3) 가격 — "이 가격에 이 성능?"              │
│                                             │
├─────────────────────────────────────────────┤
│                                             │
│  편집 스타일                                 │

│  시연훅 → 기능x3 → 가격반전 CTA              │

```

#### 컷 뷰 탭 — 타임라인형 (WakaLab 고유)

```
0s     3s     6s     9s     12s    15s
|------|------|------|------|------|
[훅  ] [소구1] [소구2] [공백] [소구3] [CTA]
시연    기능    비교           가격    구매

[썸네일][썸네일][썸네일][썸네일][썸네일][썸네일]

컷1: "이거 보세요" (0~1.8초)
컷2: "자외선 차단율이..." (1.8~3.5초)
...
```

#### 시나리오

**크리에이터 — "이 떡상 영상 왜 떴지?"**
1. 레이더에서 떡상 영상 [해킹하기] 클릭 (또는 직접 URL/업로드)
2. 해킹 결과 탭: 훅 유형, 소구 포인트, 편집 스타일, 레시피 확인
3. 더 자세히 보고 싶으면 컷 뷰 탭
4. "이 구조로 만들어야겠다" → [대본 생성] 클릭 → 대본 메뉴로

**마케터 — "우리 영상 구조 체크"**
1. 우리 팀 영상 업로드
2. 해킹 결과 확인: "훅이 질문형이네, 소구가 2개뿐이네"
3. 경쟁사 영상이랑 비교하고 싶다 → [비교에 추가] 클릭 → 비교 메뉴로

#### 기술 요구사항
- 현재 14단계 파이프라인 활용 (백엔드 변경 최소화)
- 프론트엔드: 카드형 레이아웃으로 전면 리뉴얼
- 한줄 요약 + 레시피 생성: Phase 7b 마케터 판결 확장
- URL 입력: 기존 URL 분석 기능 활용
- 개발 우선순위: 1차 (프론트엔드 리뉴얼)

### 4-3. 대본 (Script)

**목적:** "이 구조로 내 제품 대본 써줘"

#### 서브메뉴

| 서브 | 기능 |
|------|------|
| **대본 생성** | 레시피 + 제품 정보 입력 → 스크립트 자동 생성 |
| **대본 보관함** | 생성한 대본 목록, 편집·복사·공유 |

```
│                                             │
│  [복사] [저장] [PDF] [소재 확장하기]          │
│                                             │
└─────────────────────────────────────────────┘
```

#### 시나리오

**크리에이터 — "이 구조로 내 선크림 대본"**
1. 해킹에서 [대본 생성] 클릭 → 대본 메뉴로 이동
2. 레시피 자동 입력됨
3. 제품 정보 입력: 선크림 / SPF50 / 2만원 / 백탁 없음
4. [대본 생성하기] 클릭
5. 컷별 대본 출력 (타임스탬프 + 화면 설명 + 나레이션)
6. [복사] 해서 촬영에 활용

**마케터 — "PD한테 전달할 기획안"**
1. 해킹 결과 기반으로 대본 생성
2. [PDF 내보내기] → PD/편집자에게 전달
3. 대본 보관함에 저장 → 팀원과 공유

#### 기술 요구사항
- LLM 호출 (레시피 JSON + 제품 정보 → 컷별 대본 생성)
- Gemini 2.5 Flash 활용 (비용 효율)
- 대본 보관: Supabase에 저장
- 개발 우선순위: 1차

### 4-4. 소재 확장 (Expand)

#### 시나리오

**크리에이터 — "같은 선크림으로 3개 더"**
1. 대본에서 소재 A(시연+기능)를 이미 만듦
2. [소재 확장] 클릭
3. 소재 B(비교형), C(감성형), D(정보형) 제안 확인
4. 소재 B 선택 → [대본 생성] → 대본 메뉴에서 상세 대본 확인
5. 3~4개 소재를 돌아가며 올려서 반응 비교

**마케터 — "A/B 테스트용 소재 기획"**
1. 해킹한 영상의 구조 기반으로 소재 확장
2. "소재 A(기능 중심) vs 소재 C(감성 중심) 으로 테스트하자"
3. 각 소재 대본 생성 → PD에게 전달

#### 기술 요구사항
- LLM 호출 (기존 레시피 + 제품 정보 → 훅/소구/CTA 조합 변형)
- 훅 유형 풀: 시연/비교/질문/충격/공감/후기/도전 등
- 소구 유형 풀: 기능/가격/비교/감성/성분/후기/한정/권위 등
- CTA 유형 풀: 가격반전/한정/후기/직접/비교 등
- 개발 우선순위: 1차

### 4-5. 비교 (Compare)

**목적:** "A와 B 뭐가 다른가?"

#### 서브메뉴

| 서브 | 기능 |
|------|------|
| **1:1 비교** | 해킹 완료된 영상 2개의 구조를 나란히 비교 |
| **그룹 비교** | 같은 제품/카테고리의 여러 영상을 한눈에 (2차) |

#### 화면 구성 — 1:1 비교

```
┌─────────────────────────────────────────────┐
│  비교                                        │
│                                             │
│  [영상 A 선택 v]        [영상 B 선택 v]      │
│                                             │
├──────────────────┬──────────────────────────┤
│                  │                          │
│  영상 A           │  영상 B                  │
│  떡상 선크림 영상  │  우리 선크림 영상         │
│                  │                          │
│  훅: 시연형       │  훅: 질문형              │
│  소구: 3개        │  소구: 2개               │
│  편집: 1.8초/컷   │  편집: 3.2초/컷          │
│  구조: 4막        │  구조: 3막               │
│                  │                          │
│  레시피:          │  레시피:                  │
│  시연→기능x3→가격 │  질문→기능x2→일반CTA     │
│                  │                          │
├──────────────────┴──────────────────────────┤
│                                             │
│  차이점 요약                                 │
│  · 훅: 시연형 vs 질문형                      │
│  · 소구 수: 3개 vs 2개 (1개 적음)            │
│  · 편집 속도: 1.8초 vs 3.2초 (1.8배 느림)    │
│  · CTA: 가격반전 vs 일반                     │
│                                             │
│  [소재 확장으로 개선하기]                      │
│                                             │
└─────────────────────────────────────────────┘
```

#### 시나리오

**마케터 — "경쟁사 vs 우리"**
1. 비교 메뉴 → 해킹 완료된 영상 목록에서 2개 선택
2. 나란히 비교: 훅/소구/편집/레시피 차이 한눈에
3. "우리가 소구도 적고 컷도 느리구나"
4. [소재 확장으로 개선하기] → 경쟁사 구조를 참고한 소재 제안

**크리에이터 — "내 영상 vs 떡상 영상"**
1. 내 영상 해킹 + 레이더에서 떡상 영상 해킹
2. 비교해서 구조 차이 확인
3. "떡상 영상은 훅이 시연형이고 나는 질문형이었네"
4. 다음 영상은 시연형으로 시도

#### 기술 요구사항
- 프론트엔드: 해킹 결과 2개를 병렬 레이아웃
- 차이점 요약: LLM 또는 룰 기반 diff
- 백엔드: 기존 해킹 결과 2개 조회 (추가 API 최소)
- 개발 우선순위: 2차

## 5. 메뉴 간 연결 흐름

```
레이더 ----> 해킹 ----> 대본 ----> 소재확장
  |            |                      ^
  |            +----> 비교 -----------+
  |                    |
  +---------------> 인사이트 (데이터 축적 후)
```

모든 메뉴는 서로 연결됨:
- 레이더에서 [해킹하기] → 해킹으로
- 해킹에서 [대본 생성] → 대본으로
- 해킹에서 [소재 확장] → 소재확장으로
- 해킹에서 [비교에 추가] → 비교로
- 대본에서 [소재 확장하기] → 소재확장으로
- 소재확장에서 [대본 생성] → 대본으로
- 비교에서 [소재 확장으로 개선하기] → 소재확장으로
