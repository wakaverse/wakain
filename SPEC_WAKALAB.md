## 2. 핵심 가치 5가지

### ① 비교 진단 (Comparative Diagnosis)
> "잘 되는 영상과 안 되는 영상, 차이가 뭔지 찾아드립니다"

- 영상 A(CTR 낮음) vs B(CTR 높음) 비교 분석
- "B에는 있고 A에는 없는 것" 도출
- 단순 진단을 넘어 '상대적 우위'를 점하게 해주는 기능
- 마케터를 가장 강하게 자극하는 지점

### ② 수정의 정석 (Actionable Feedback)
> "다시 찍지 마세요. 어디를 왜 고쳐야 하는지 알려드립니다"

- 이탈 구간 + 원인 진단 (소구 부적합, 시각적 자극 부족 등)
- 소구 포인트 교체 제안 ("공포 마케팅 → 사회적 증거로 바꾸세요")
- **효과**: 제작비 낭비 방지, ROAS 빠른 최적화

### ③ 성공 공식 자산화 (Creative Intelligence)
> "당신 브랜드만의 승리 방정식을 데이터로 만듭니다"

- 같은 브랜드 영상 N개 분석 → 패턴 발견
- "텍스트 오버레이보다 내레이션 강조 시 CTR 1.5배"
- 담당자가 바뀌어도 유지되는 브랜드 퍼포먼스 가이드라인
- **효과**: 조직 지식 축적, 인수인계 비용 제거

### ④ 기획 자동화 (Scalable Planning)
> "다음엔 뭐 찍지? 검증된 구조로 기획안을 뽑아드립니다"

- 씬 구조(Hook→Demo→Proof→CTA) 기반 기획안 생성
- 카테고리별 잘 팔리는 구조에 새 제품만 교체
- A/B 테스트 설계: "Hook을 3가지 소구로 변형"
- → WakaShorts 연결 시 기획→제작 원스톱
- **효과**: 기획 시간 80% 단축, 소재 다변화 가속

### ⑤ 객관적 소통 (Standard Language)
> "'느낌 있게 만들어주세요'가 아니라 수치로 소통하세요"

- 컷 밀도 초당 1.2회, 아트 디렉션 톤 파스텔, 폰트 고딕
- 마케터 ↔ PD/편집자 사이 언어 통일
- **효과**: 수정 피드백 루프 감소, 협업 효율 극대화

## 3. 결과물 (Output)

### 핵심 철학
> **"왜 안 되는지"에 대한 집요한 추적**
> → 팩트 폭격 + 해결책 동시 제공

### 기초 결과물 (Core)

#### 결과물 1: 소구 흐름도 (Appeal Flow) — "발견"
> 영상의 설득 구조를 한 눈에 → 약점을 시각적으로 즉시 파악

- 컷별 소구 포인트 매핑 (18종)
- 소구 공백 구간 하이라이트
- 씬 역할 표시 (hook/demo/proof/recap/cta)
- 시간축 기반 시각화

```
시간  컷    역할     소구              내용
───────────────────────────────────────────
0s   컷1   hook     [myth_bust]       "김이 다 거기서 거기라고요?"
3s   컷2   demo     [sensory]         밥 위에 김 클로즈업
5s   컷3   demo     [specification]   4가지 맛 제품 나열
7s   컷4   proof    [social_proof]    일본 수출 1000만 달러
14s  컷7   ──       [── 공백 ──]      ⚠️ 소구 없음 → 이탈 위험
18s  컷9   cta      [guarantee]       맛 없으면 보장
```

#### 결과물 2: 씬 카드 (Scene Cards) — "근거"
> "왜 이 장면이 효과적인가" — 제작팀과의 소통 근거

- 컷별 상세 분석 (썸네일 + 소구 + 시각 + 편집 + 내레이션/캡션)
- "이 장면이 효과적인 이유" 설명
- 수치와 실사 이미지를 결합한 설명

```
┌─────────────────────────────────┐
│ 컷 4 (7~10s) — proof            │
│ 🎬 일본 마트 진열대 실사 이미지   │
│ 🗣️ "천만 달러 이상 수출되고..."   │
│ 📝 "1000만딸라" 텍스트 오버레이   │
│ 소구: [social_proof] 🟢 strong   │
│                                 │
│ 💡 효과적인 이유:                │
│   수치 + 실사 이미지 = 신뢰감     │
└─────────────────────────────────┘
```

#### 결과물 3: 진단서 (Diagnosis) — "처방"
> 강점/약점을 넘어 구체적 수정 가이드 — 리포트의 백미

```
✅ 강점
• Hook 0초에 myth_bust로 즉시 호기심 유발
• 소구 다양성 9종 — 다각도 설득
• proof 구간에 3연속 소구 — 신뢰 축적 구조

⚠️ 약점  
• 14~16s 소구 공백 — 설득 흐름 끊김
• CTA가 갑작스러움 — 전환 장치 부족

💊 처방
• 14s 공백에 [social_proof] 리뷰 장면 삽입
• CTA 진입 전 [price] + [urgency] 소구 추가
• 컷 7을 가격/한정 수량으로 교체 권장
```

### 확장 결과물 (Premium)

#### 결과물 4: 비교 리포트 (Compare)
> A vs B — 뭐가 다른지 + 처방전

```
        영상 A (안 됨)          영상 B (잘 됨)
hook    [lifestyle] 약함        [myth_bust] 강함
proof   없음 ❌                [social_proof] 3개 ✅
공백    2구간                   0구간

→ 처방: A의 hook을 myth_bust로 교체,
  6s에 리뷰 삽입, CTA에 보장 멘트 추가
```

#### 결과물 5: 아트 디렉션 가이드
> 비주얼 스타일 레퍼런스

- 톤앤매너, 폰트, 컬러, 강조 방식
- 텍스트 위치 패턴, 프레임 구성 규칙
- 제작팀 전달용 비주얼 가이드라인

#### 결과물 6: 다음 영상 기획안 (Production Guide)
> 분석 기반 검증된 구조의 설계도

```
추천 구조 (30초):
  0~3s:  [myth_bust] hook
  3~8s:  [feature_demo] 제품 시연
  8~12s: [social_proof] 리뷰/수치 3개
  12~16s:[manufacturing] 제조 차별점
  16~20s:[price] + [urgency]
  20~30s:[guarantee] CTA

→ WakaShorts로 제작하기
```

## 5. 알고리즘 고도화

### 5.1 STT 분리 + 영상 유형 분기

**현재:** Gemini가 STT + 분석 한번에 (부정확)
**개선:** STT 전용 API → 유형 판단 → 평가 분기

```
Phase 0 (신규): STT 전용 API
  ├── 로컬: Whisper (설치됨, ₩0)
  └── 배포: Google Cloud STT
        ↓
  음성 있음 → narration_type = "voice"
  음성 없음 → narration_type = "caption"
```

#### Track A: 내레이션 영상 (voice)
- 주요 분석: **음성 텍스트** (STT)
- 내레이션 흐름으로 설득 구조 분석
- 캡션은 보조 역할
- Phase 4 프롬프트: 내레이션 기반 소구 분석

#### Track B: 캡션 전용 영상 (caption)
- 주요 분석: **텍스트 오버레이** (OCR)
- 캡션 시간순 정렬 → 스토리 흐름 재구성
- 타이포그래피 분석 강화
- BGM/효과음 역할 증대
- Phase 4 프롬프트: 텍스트 오버레이 기반 소구 분석

### 5.2 캡션 시간순 스토리 매핑

**현재:** 프레임별 OCR → 텍스트 조각들
**개선:** 시간순 정렬 → 설득 흐름 재구성

```
0s: "매일 아침 피곤하시죠?"     → hook (문제 제기)
3s: "이 성분이 다릅니다"        → solution
6s: "임상시험 결과 89% 개선"    → proof
9s: "지금 50% 할인"            → cta
```

캡션 = 내레이션의 대체재 → Track B의 핵심 입력

### 5.3 시각 자극도 커브 (Visual Stimulus Curve)

> 커브(그래프)는 보여줌 / 숫자 점수는 보여주지 않음
> 소구 흐름도의 배경 레이어로 활용

- 시각 자극도 커브 = 진단 차트의 **배경**
- 소구 흐름 = 커브 **위에 올리는 레이어**
- 커브 + 소구가 겹칠 때 진짜 인사이트 발생:
  - 소구 있는데 커브 낮음 → "이 소구의 시각 표현이 약하다"
  - 소구 공백 + 커브 하락 → "진짜 이탈 위험 구간"
  - 소구 없는데 커브 높음 → "시각은 좋은데 설득이 없다"

```
시각 자극도  ████████████▁▁▁▁████████
소구 흐름    [myth] [sensory]  ←공백→  [proof] [cta]
                              ↑
                    커브 하락 + 소구 공백 = ⚠️ 위험
```

### 5.4 다차원 분석 (내부 로직)

> 마케터에게 종합 점수를 보여주지 않음
> 내부적으로만 이탈 예측/진단의 근거로 활용

| 차원 | 데이터 소스 | 측정 내용 |
|------|-----------|----------|
| 시각적 자극도 | OpenCV | 색상/밝기/모션 변화량 |
| 설득 밀도 | Phase 4 소구 데이터 | 소구 유무, 강도, 공백 |
| 정보 밀도 | OCR + STT | 텍스트양, 내레이션 속도 |
| 편집 리듬 | PySceneDetect | 컷 밀도, 전환 빈도 |
| 청각 자극도 | Phase 4 프롬프트 확장 | BGM 변화, 효과음 |

#### 영상 유형별 내부 가중치

| 차원 | Track A (내레이션) | Track B (캡션) |
|------|-------------------|----------------|
| 시각적 자극도 | 20% | 30% |
| 설득 밀도 | 35% | 25% |
| 정보 밀도 | 25% | 30% |
| 편집 리듬 | 10% | 10% |
| 청각 자극도 | 10% | 5% |

### 5.4 소구 포인트 = 분석의 핵심 드라이버

소구 포인트는 하위 차원이 아니라 **모든 차원을 관통하는 핵심 축**:
- 소구가 있는 구간: 시각+설득+정보+편집+청각 모두 활성화
- 소구가 빠진 구간: 모든 차원 동반 하락 → 이탈 위험

### 5.5 이탈 예측 고도화

| 항목 | 기존 | 변경 |
|------|------|------|
| 시각적 자극도 저하 | < 20, 3초 이상 | < 40, 2초 이상 |
| 소구 공백 | 5초 이상 | 3초 이상 |
| 씬 정체 | 4초 이상 | 3초 이상 |
| 상대적 급락 (신규) | 없음 | 직전 2초 대비 급락 시 |
| 종합 계산 | 위험요소만 반영 | 내부 Engagement 점수 가중 |

### 5.6 영상 스타일별 분석 엔진

#### 2축 분류 체계: 형식(Format) × 의도(Intent)

> 같은 진행자형이어도 커머스 vs 브랜딩이면 평가 기준이 완전히 다름
> → 형식과 의도를 분리하여 2축으로 분류

**형식 (Format) — 영상이 어떻게 만들어졌는가:**

| 형식 | 특징 |
|------|------|
| 진행자형 | 사람이 말하며 설명 |
| UGC/브이로그형 | 실사용자 리얼 후기 |
| 캡션/텍스트형 | 무음 + 텍스트만 |
| 데모/언박싱형 | 제품 기능 시연 |
| ASMR/감성형 | 시각+청각 감성 자극 |
| 비교형 | A vs B 직접 비교 |
| 스토리/문제해결형 | 문제→해결 서사 |
| 나열형 | 제품 여러 개 소개 |
| 엔터테인먼트형 | 반전/유머 중심 |

**의도 (Intent) — 영상이 무엇을 달성하려는가:**

| 의도 | 목적 |
|------|------|
| 커머스/세일즈 | 즉각적인 구매 유도 |
| 브랜딩/이미지 | 브랜드 인지도 |
| 정보 전달 | 교육/설명 |
| 엔터테인먼트 | 바이럴/공유 |

#### 사용자 선택 + Auto 모드

```
업로드 화면:

영상 형식:  [Auto ▼]  ← 기본값
영상 의도:  [Auto ▼]  ← 기본값

→ 둘 다 Auto면 영상만 던지면 끝
→ 알면 선택, 모르면 Auto
→ 경쟁사 영상 10개 분석할 때 일일이 선택할 필요 없음
```

**Auto 모드 판별:**
- Gemini 저비용 모델(Flash Lite)로 형식+의도 자동 판별
- 결과를 리포트 상단에 표시
- "분류가 틀렸나요? [수정하기]" → 재분석 가능

**복합 스타일 처리:**
- 주 형식 + 보조 형식으로 분류 (예: 진행자형 + 데모형)
- 사용자가 직접 복합 선택도 가능

#### 3단계 통합 분석 프로세스

```
1단계: 입체적 분류 (Classification)
  형식: 진행자형 + 데모형
  의도: 커머스/세일즈형

2단계: 동시 고려 분석 (Simultaneous Analysis)
  추출: '진행자형' → 내레이션 STT + 발표자 표정 정밀 분석
       '데모형' → 제품 클로즈업 샷 타임라인 기록
  평가: '커머스형' → 첫 소구 등장 시간(0.0초), 컷 밀도(0.55) 긍정 평가
       만약 '브랜딩형'이었다면 컷 밀도가 너무 높다고 경고

3단계: 통합 진단 및 처방 (Integrated Diagnosis)
  진단: "진행자형 데모 영상치고는 14~16초 구간에 
        제품 시각 자극(데모)이 비어 있어 집중도가 떨어짐"
  처방: "커머스 성과를 높이려면 해당 구간에 
        제품의 바삭함을 보여주는 [sensory] 클로즈업 샷을 추가하세요"
```

#### 스타일별 핵심 지표 우선순위

| 스타일 | 핵심 지표 | 덜 중요한 지표 |
|--------|----------|---------------|
| **커머스/세일즈형** | 첫 소구 등장 시간, 컷 밀도 | 색감 일관성 |
| **UGC/브이로그형** | 발표자 표정, 내레이션 진정성 | 컷 밀도 (빠르면 오히려 가짜 느낌) |
| **브랜딩/이미지형** | 아트 디렉션 일관성, 색조 | 수치적 자극도 |
| **ASMR/감성형** | 청각 자극도, 색감/무드 | 정보 밀도 |
| **데모형** | 기능 시연 순서, 클로즈업 빈도 | 내레이션 흐름 |

#### 스타일별 유효 소구 포인트

| 스타일 | 강한 소구 | 약한 소구 |
|--------|----------|----------|
| **정보 전달형** | myth_bust, specification, spec_data | sensory, nostalgia |
| **감성 소구형** | sensory, lifestyle, design_aesthetic | track_record, spec_data |
| **커머스형** | price, social_proof, guarantee | nostalgia, authority |
| **UGC형** | authenticity, social_proof, lifestyle | specification, comparison |
| **브랜딩형** | design_aesthetic, emotional, authority | price, urgency |

#### 스타일별 구조 변형

| 스타일 | 권장 구조 |
|--------|----------|
| **커머스/세일즈형** | Hook → Demo → Proof → CTA (정석) |
| **문제 해결형** | Problem(공감) → Solution(해결) → Proof → CTA |
| **엔터테인먼트형** | Hook → Twist → Twist → Twist → CTA (반전 반복) |
| **UGC형** | 일상 → 발견 → 체험 → 솔직 후기 |
| **브랜딩형** | 무드 → 가치 전달 → 브랜드 각인 |
| **비교형** | Before/기존 → After/신제품 → 차이 강조 → CTA |

#### 스타일별 처방 사전 (Style-Specific Prescriptions)

> **같은 현상이어도 스타일에 따라 처방이 반대 — WakaLab의 핵심 차별점**

| 현상 | 커머스형 처방 | UGC형 처방 |

| 아트디렉션 불일관 | 🟡 "통일하면 좋음" | ✅ "리얼리티 느낌" |
| BGM 없음 | 🔴 "BGM 추가 권장" | ✅ "오히려 진정성 강화" |
| 텍스트 많음 | ✅ "정보 전달에 효과적" | ⚠️ "광고 느낌 날 수 있음" |
